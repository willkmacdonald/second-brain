Create `backend/scripts/migrate_shopping_to_errands.py` -- a one-time migration script that:

1. Uses `azure.cosmos.aio.CosmosClient` with `azure.identity.aio.DefaultAzureCredential` (same pattern as CosmosManager)
2. Reads `COSMOS_ENDPOINT` from environment variable (required)
3. Database name: `"second-brain"` (hardcoded, matches codebase)
4. Creates the `Errands` container with partition key `/destination` using `create_container_if_not_exists`
5. Iterates over known destinations: `["jewel", "cvs", "pet_store", "other"]`
6. For each destination, queries all items from `ShoppingLists` container using `partition_key=destination`
7. For each item, creates a new document in `Errands` with:
   - `id`: same as source
   - `name`: same as source
   - `destination`: value of source `store` field
8. Counts items migrated per destination and total
9. After migration, verifies total count in `Errands` matches items migrated
10. If verification passes, deletes the `ShoppingLists` container using `database.delete_container("ShoppingLists")`
11. Prints summary: items migrated per destination, total, verification result
12. Uses proper async cleanup: close client and credential in finally block

**Error handling:**
- If `COSMOS_ENDPOINT` not set, print error and exit with code 1
- If `ShoppingLists` container doesn't exist (already migrated), print message and exit cleanly
- If any item copy fails, log error but continue with remaining items
- If verification count doesn't match, DO NOT delete old container -- print warning and exit with code 1
- Wrap entire script in `asyncio.run(migrate())`

**Script should be runnable as:** `python3 backend/scripts/migrate_shopping_to_errands.py`
