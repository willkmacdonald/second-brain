## Summary

This phase addresses 12 security issues (1 critical, 6 important, 5 medium) and 12 dead code findings documented in `CONCERNS-2.md`. All findings have been verified against the current codebase -- every file, line number, and code pattern matches exactly. The fixes are straightforward, well-scoped, and require no new dependencies or architectural changes.

**Primary recommendation:** Group work into 3 plans: security fixes (C1/I1-I6), input validation + dead code cleanup (M1-M5/D1-D12), and duplicate consolidation (D9/D10). Each plan is independently deployable and testable.

## Architecture Patterns

### Pattern 1: Timing-Safe API Key Comparison (C1)

**What:** Replace `!=` with `hmac.compare_digest()` in auth middleware.
**When to use:** Any secret comparison in auth code.
**Verified location:** `backend/src/second_brain/auth.py:61`

Current code:
```python
if api_key is None or provided_key != api_key:
```

Fix:
```python
import hmac
if api_key is None or not hmac.compare_digest(provided_key, api_key):
```

**Confidence:** HIGH -- this is a textbook CWE-208 fix. Python docs explicitly recommend `hmac.compare_digest()` for this purpose.

### Pattern 2: Parameterized Cosmos DB Queries (I1)

**What:** Replace f-string interpolated queries with `@parameter` syntax in `admin.py`.
**When to use:** Any Cosmos DB query that includes user-controllable or LLM-sourced values.
**Verified locations:** `backend/src/second_brain/tools/admin.py:254,273,320` (all in `_destination_create`, `_destination_rename`, `_destination_remove`)

The `_collect_query` helper currently accepts only `query` and `partition_key`. It needs to be extended to accept an optional `parameters` argument and pass it through to `container.query_items()`. The Cosmos DB `query_items` API accepts a `parameters` keyword argument as `list[dict[str, Any]]`.

Current code (repeated 3 times):
```python
f"SELECT * FROM c WHERE c.slug = '{slug}' AND c.userId = 'will'"
```

Fix pattern:
```python
query = "SELECT * FROM c WHERE c.slug = @slug AND c.userId = 'will'"
parameters = [{"name": "@slug", "value": slug}]
```

All other Cosmos queries in the project already use parameterized queries correctly (verified in `inbox.py`, `errands.py`, `cosmos_crud.py`). The `admin.py` f-string queries are the sole outliers.

**Confidence:** HIGH -- the parameterized query pattern is already established throughout the codebase.

### Pattern 3: Error Message Sanitization (I2, I3, M5)

**What:** Replace raw `str(exc)` in client-facing error responses with generic messages; log full exception server-side.
**Verified locations:**
- `adapter.py:312,494,631` -- SSE error events stream `str(exc)` to mobile client
- `classification.py:126` -- `file_capture` returns `{"detail": str(exc)}`
- `cosmos_crud.py:74,99,132` -- dead code (D1), will be deleted
- `transcription.py:91` -- returns `f"Transcription error: {exc}"`

The project already follows the pattern of `logger.error()` + generic user message in most places. The flagged locations are exceptions where raw exceptions leak.

Fix pattern:
```python
# Before
yield encode_sse(error_event(str(exc)))

# After
logger.error("Capture stream error: %s", exc, exc_info=True)
yield encode_sse(error_event("An internal error occurred. Please try again."))
```

For tool error returns (classification.py, transcription.py), the "client" is the LLM agent, not the mobile user directly. However, agent responses can be echoed to the user. Generic messages are safer:
```python
# classification.py
return {"error": "cosmos_write_failed", "detail": "Failed to save classification"}

# transcription.py
return "Transcription failed. Please try again."
```

**Confidence:** HIGH -- pattern already exists in the codebase; these are gaps.

### Pattern 4: Upload Guardrails (I4, I5)

**What:** Add file size and content type validation to voice capture endpoints.
**Verified locations:** `backend/src/second_brain/api/capture.py:201,314` (file.read()), `capture.py:184,296` (no content_type validation)

For size validation, check `file.size` (set by FastAPI/Starlette from Content-Length header) before reading, and also limit bytes read. 25 MB is the standard limit for audio transcription APIs.

For content type validation, check against allowed MIME types. The mobile app sends `audio/m4a` files.

```python
MAX_AUDIO_SIZE = 25 * 1024 * 1024  # 25 MB
ALLOWED_AUDIO_TYPES = {"audio/m4a", "audio/mp4", "audio/mpeg", "audio/wav", "audio/x-m4a"}

# Size check
if file.size and file.size > MAX_AUDIO_SIZE:
    raise HTTPException(status_code=413, detail="Audio file too large (max 25 MB)")

# Content type check
content_type = file.content_type or ""
if content_type not in ALLOWED_AUDIO_TYPES:
    raise HTTPException(
        status_code=415,
        detail=f"Unsupported audio format: {content_type}",
    )

audio_bytes = await file.read()
if len(audio_bytes) > MAX_AUDIO_SIZE:
    raise HTTPException(status_code=413, detail="Audio file too large (max 25 MB)")
```

Note: `file.size` may be `None` if Content-Length header is absent (streaming upload). The post-read check is the safety net. Both voice endpoints (`capture_voice` at line 184 and `follow_up_voice` at line 296) need identical validation.

**Confidence:** HIGH -- straightforward validation. Both endpoints share the same pattern so a helper function is appropriate.

### Pattern 5: Disable OpenAPI Docs in Production (I6)

**What:** Remove `/docs` and `/openapi.json` from `PUBLIC_PATHS` when running in production.
**Verified location:** `backend/src/second_brain/auth.py:19`

The cleanest approach is adding an `ENVIRONMENT` setting to `config.py` and conditionally setting FastAPI's `docs_url` and `openapi_url` to `None` in production. This disables the routes entirely rather than requiring auth for them.

```python
# config.py
environment: str = "development"  # "development" or "production"

# main.py
settings = get_settings()
docs_url = "/docs" if settings.environment == "development" else None
openapi_url = "/openapi.json" if settings.environment == "development" else None
app = FastAPI(title="Second Brain API", lifespan=lifespan, docs_url=docs_url, openapi_url=openapi_url)
```

Then remove `/docs` and `/openapi.json` from `PUBLIC_PATHS` in `auth.py` since they no longer exist in production.

Alternative: read `ENVIRONMENT` directly from `os.environ` in auth.py to avoid import circularity. However, FastAPI natively supports disabling docs via constructor args, which is cleaner.

Note: `settings = get_settings()` cannot be called at module level before `load_dotenv()` runs. The `docs_url`/`openapi_url` must be determined after env vars are loaded. Since `main.py` calls `load_dotenv()` first and then creates the app, this works if the app creation is moved slightly or done conditionally.

**Confidence:** HIGH -- FastAPI's `docs_url=None` pattern is documented and canonical.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Timing-safe comparison | Custom byte-by-byte compare | `hmac.compare_digest()` | Stdlib, audited, constant-time guaranteed |
| SQL injection prevention | Custom escaping/sanitization | Cosmos DB `@parameters` | Cosmos SDK handles escaping; hand-rolled escaping will miss edge cases |
| File upload validation | Custom streaming size limiter | `file.size` + post-read length check | FastAPI/Starlette already parses Content-Length; double-check is sufficient |

**Key insight:** Every fix in this phase uses existing patterns already established in the codebase. The goal is consistency, not invention.

## Common Pitfalls

### Pitfall 1: Breaking the `_collect_query` Helper Signature

**What goes wrong:** Adding `parameters` to `_collect_query` but forgetting to pass it through, or changing the signature in a way that breaks the 9 existing callers that don't use parameters.
**Why it happens:** The helper is used throughout `admin.py` -- most callers don't need parameters.
**How to avoid:** Make `parameters` optional with `None` default. Only pass to `query_items()` when not `None`.
**Warning signs:** Any call to `_collect_query` that doesn't compile.

### Pitfall 2: Forgetting the Post-Read Size Check

**What goes wrong:** Only checking `file.size` (from Content-Length header) but a client omits the header and uploads a huge file.
**Why it happens:** `file.size` is `None` when Content-Length is absent.
**How to avoid:** Always check `len(audio_bytes)` after `await file.read()` as a safety net.
**Warning signs:** `file.size` is `None` in logs.

### Pitfall 3: Module-Level Settings Access for Docs Toggle

**What goes wrong:** Calling `get_settings()` at module level in `main.py` before `load_dotenv()` has run.
**Why it happens:** Python evaluates module-level code at import time; `load_dotenv()` runs inside the module body of `main.py`.
**How to avoid:** The `app = FastAPI(...)` line is already in module-level scope but comes AFTER `load_dotenv()`. Just use `get_settings()` in the same scope, or use `os.environ.get("ENVIRONMENT", "development")` directly.
**Warning signs:** Settings returning empty strings or defaults.

### Pitfall 4: Deleting Dead Code That Tests Reference

**What goes wrong:** Deleting `cosmos_crud.py` but leaving `test_cosmos_crud.py`, causing test failures.
**Why it happens:** Forgetting to delete the test file.
**How to avoid:** Always delete the test file alongside the dead module.
**Warning signs:** `ImportError` in test suite.

### Pitfall 5: Consolidating VALID_BUCKETS Breaks Import Order

**What goes wrong:** Moving `VALID_BUCKETS` to `documents.py` but creating a circular import because `classification.py` imports from `documents.py` which might import from classification.
**Why it happens:** Import cycles in Python.
**How to avoid:** `documents.py` has no imports from other `second_brain` modules (verified). Adding a constant there is safe.
**Warning signs:** `ImportError: cannot import name 'VALID_BUCKETS'` at startup.

### Pitfall 6: Consolidating Routing Context Breaks Formatting

**What goes wrong:** Merging `admin_handoff._build_routing_context` and `admin.get_routing_context` but the two have slightly different formatting (e.g., "AVAILABLE DESTINATIONS:" vs "DESTINATIONS:").
**Why it happens:** They were written at different times for different consumers.
**How to avoid:** Use the `admin.py` tool version as the canonical one (it's the one the Admin Agent actually calls). Update `admin_handoff.py` to call the tool's method or a shared function with the tool's format.
**Warning signs:** Admin Agent routing quality regresses because context format changed.

### Extending _collect_query for Parameters (I1)

```python
async def _collect_query(
    self,
    container_name: str,
    query: str,
    partition_key: str = "will",
    parameters: list[dict] | None = None,
) -> list[dict]:
    """Run a Cosmos query and collect all results into a list."""
    container = self._manager.get_container(container_name)
    results: list[dict] = []
    kwargs: dict = {"query": query, "partition_key": partition_key}
    if parameters is not None:
        kwargs["parameters"] = parameters
    async for item in container.query_items(**kwargs):
        results.append(item)
    return results
```

### Audio Upload Validation Helper (I4, I5)

```python
MAX_AUDIO_SIZE = 25 * 1024 * 1024  # 25 MB
ALLOWED_AUDIO_TYPES = frozenset({
    "audio/m4a", "audio/mp4", "audio/mpeg", "audio/wav",
    "audio/x-m4a", "audio/aac", "audio/ogg",
})

async def _validate_and_read_audio(file: UploadFile) -> bytes:
    """Validate audio upload and return bytes. Raises HTTPException on failure."""
    content_type = file.content_type or ""
    if content_type not in ALLOWED_AUDIO_TYPES:
        raise HTTPException(
            status_code=415,
            detail=f"Unsupported audio format: {content_type}",
        )
    if file.size and file.size > MAX_AUDIO_SIZE:
        raise HTTPException(status_code=413, detail="Audio file too large (max 25 MB)")
    audio_bytes = await file.read()
    if len(audio_bytes) > MAX_AUDIO_SIZE:
        raise HTTPException(status_code=413, detail="Audio file too large (max 25 MB)")
    return audio_bytes
```
