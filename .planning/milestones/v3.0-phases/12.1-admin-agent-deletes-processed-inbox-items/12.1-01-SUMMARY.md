## Decisions Made
- Processing triggers as side effect of GET /api/shopping-lists (not a separate endpoint) -- aligns with user workflow where opening Status screen initiates processing
- Delete-on-success replaces processed upsert -- shopping list items are the durable output, inbox items are transient
- CosmosResourceNotFoundError on delete is handled gracefully (user may have swipe-deleted while processing was running)
- 3-second polling interval for auto-refresh while processing is active -- balances responsiveness with API load
- processingCount field added to ShoppingListResponse for mobile banner control
