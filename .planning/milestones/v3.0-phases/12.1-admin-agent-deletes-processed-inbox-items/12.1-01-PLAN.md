# Plan 12.1-01: Remove Auto-Trigger, Add User-Initiated Processing, Delete on Success

<objective>
Change the Admin Agent processing trigger from automatic (fires immediately after classification) to user-initiated (fires when user opens the Status screen). After successful processing, delete the inbox item instead of flagging it as "processed". Failed items remain in Inbox with `adminProcessingStatus: "failed"` and auto-retry on next Status screen open.

Purpose: Give the user a review step in Inbox before processing happens, and keep Inbox clean by deleting successfully processed items.
Output: Modified backend endpoints, updated admin_handoff logic, mobile UI indicators.
</objective>

**CONTEXT.md locked decisions (NON-NEGOTIABLE):**
- Remove the automatic `asyncio.create_task` call in adapter.py
- Processing triggers when the user opens the Status screen -- GET /api/shopping-lists checks for unprocessed Admin inbox items and kicks off processing as a side effect
- All unprocessed Admin items (no `adminProcessingStatus` or `adminProcessingStatus: "failed"`) are processed at once
- Once triggered, processing continues in background even if user navigates away
- Admin Agent processes whatever the current rawText is (user may have edited in Inbox)
- Inbox item is deleted after successful processing
- If Admin Agent fails, inbox item stays with `adminProcessingStatus: "failed"`
- If delete fails after processing succeeds, log warning and move on

**User's workflow:** capture -> lands in Inbox -> user reviews/edits -> opens Status -> processing triggers -> shopping list items appear -> inbox item deleted

            # Delete inbox item after successful processing
            try:
                await inbox_container.delete_item(
                    item=inbox_item_id, partition_key="will"

                    "Deleted processed inbox item %s",
                    inbox_item_id,
                )

                # User may have swipe-deleted while we were processing -- that's fine

                    del_exc,
                )

                "Admin Agent processed inbox item %s: %s",
                inbox_item_id,
                response.text[:100] if response.text else "(no text)",
