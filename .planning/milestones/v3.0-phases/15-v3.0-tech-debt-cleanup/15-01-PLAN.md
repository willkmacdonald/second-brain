  **1. Fix recipe tool tests in test_recipe_tools.py:**

  All 7 tests in `TestFetchRecipeUrl` (lines 146-234) call `tools.fetch_recipe_url()` without mocking the `_fetch_jina` and `_fetch_simple` methods. This causes real HTTP calls to `r.jina.ai` which either succeed with unexpected content or fail with SSL errors, making assertions fail.

  For EVERY `async def test_*` method in `TestFetchRecipeUrl`, wrap the `await tools.fetch_recipe_url(...)` call in `patch.object` mocks that force the code path to reach Playwright (tier 3):

  ```python
  from unittest.mock import patch

  # Inside each test method, after creating the tools instance:
  with (
      patch.object(tools, "_fetch_jina", return_value=""),
      patch.object(tools, "_fetch_simple", return_value=("", "", "mock")),
  ):
      result = await tools.fetch_recipe_url(url="https://example.com/recipe")
  ```

  Apply this pattern to all 7 tests:
  - `test_successful_fetch_returns_page_text` (line 146)
  - `test_successful_fetch_with_json_ld` (line 156)
  - `test_fetch_failure_returns_error_string` (line 177)
  - `test_context_closed_on_success` (line 189)
  - `test_context_closed_on_error` (line 199)
  - `test_empty_page_returns_error` (line 211)
  - `test_text_truncated_to_limit` (line 224)

  For `test_fetch_failure_returns_error_string` and `test_context_closed_on_error`, note that `_fetch_jina` returning "" will cause the code to try `_fetch_simple`. Since `_fetch_simple` is also mocked to return empty text, the code path falls through to Playwright where the `goto_side_effect=TimeoutError(...)` is set up. The mock should work as-is.
