  <how-to-verify>
    **Prerequisites:**
    - Install the new development build from EAS on your iPhone (iOS 26.3.1)
    - Backend deployed to brain.willmacdonald.com (push backend changes to main first if not already)
    - Start Metro: `cd mobile && npx expo start --clear`

    **Test 1: On-device voice capture (happy path)**
    1. Open the app, ensure Voice mode is selected
    2. Grant speech recognition permission when prompted (first time only)
    3. Tap the record button
    4. Say: "need milk and eggs from Jewel"
    5. Expected: Words appear on screen in real-time as you speak (like iMessage dictation)
    6. Expected: Pulsing red ring around record button during recording
    7. Tap stop
    8. Expected: Transcribed text stays visible, "Classifying..." appears below (no "Uploading..." step)
    9. Expected: Filed toast appears (e.g., "Filed -> Admin (0.85)")
    10. Verify in Status tab that the errand item appeared

    **Test 2: Multi-word informal capture**
    1. Record: "remind me to call the vet about the cat's appointment"
    2. Expected: Real-time text appears as you speak
    3. Expected: Classification completes (likely People or Admin bucket)

    **Test 3: Quick note capture**
    1. Record: "check if the new iPhone case fits"
    2. Expected: Transcription captures the informal note accurately
    3. Expected: Classification completes

    **Test 4: Voice follow-up (HITL)**
    1. Record something ambiguous that triggers a misunderstood response
    2. When agent asks a follow-up question, switch to voice mode if not already
    3. Record your follow-up reply
    4. Expected: Follow-up text transcribes on-device and submits as text follow-up

    **Test 5: Empty/very short recording**
    1. Tap record, immediately tap stop (< 1 second)
    2. Expected: Graceful handling (either submits empty or discards silently)

    **Accuracy check (VOICE-OD-02):**
    - Transcriptions should be good enough for informal captures (shopping items, reminders, quick notes)
    - Minor word errors are acceptable — the classifier can usually handle them
    - If accuracy is consistently poor, report specific examples
