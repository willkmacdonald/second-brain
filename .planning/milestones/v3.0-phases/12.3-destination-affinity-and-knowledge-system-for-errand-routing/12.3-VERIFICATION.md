**Test:** Say "pick up salmon" (assuming no affinity rule for salmon) into the capture screen, wait for processing, then open the Status screen.
**Expected:** A "Needs Routing" section appears with "salmon" listed and a horizontal row of destination chips below it.
**Why human:** Requires live Admin Agent routing with no matching affinity rule, seeded Destinations container, and the mobile UI rendering the unrouted section with RoutingPicker.

#### 3. HITL Routing Chip Interaction

**Test:** From the Needs Routing section, tap a destination chip (e.g. "Jewel-Osco").
**Expected:** The item disappears from Needs Routing immediately (optimistic remove), and after a brief moment the Status screen refreshes showing the item in the Jewel-Osco section.
**Why human:** Requires a running mobile app against the live backend and an actual POST /api/errands/{id}/route call.

#### 4. Online Destination Visual Indicator

**Test:** After running the seed script (which creates Chewy as type="online"), open the Status screen with at least one item in the Chewy destination.
**Expected:** The Chewy section header shows a small "ONLINE" pill/tag in blue next to the destination name. Physical destinations (Jewel-Osco, CVS) have no such tag.
**Why human:** Visual rendering requires a real device or simulator with the app running.

#### 5. Rule Query Answer Delivered as Notification

**Test:** Say "where does chicken go?" into the capture screen (after creating a rule for chicken), wait for processing, open the Status screen.
**Expected:** A dismissible notification banner appears showing the Admin Agent's rule query answer.
**Why human:** Requires Admin Agent instructions that route query-type captures to query_rules tool, and the _response_needs_delivery heuristic to keep the completed inbox item for delivery.
