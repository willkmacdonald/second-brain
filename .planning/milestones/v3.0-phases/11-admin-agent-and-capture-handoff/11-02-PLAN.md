<objective>
Wire the fire-and-forget Admin Agent trigger into the capture flow so that Admin-classified captures are silently processed in the background.

Purpose: Connect the process_admin_capture function (from Plan 11-01) to the live capture flow. When the Classifier files a capture with bucket=="Admin", the adapter spawns a background task to run the Admin Agent. The SSE stream closes immediately -- the user never waits for or sees Admin Agent processing. This completes the core capture-to-shopping-list pipeline.

Output: Modified adapter.py, capture.py, main.py; verified end-to-end flow

**Key design decisions per CONTEXT.md:**
- Fire-and-forget: `asyncio.create_task` -- no `await`, SSE stream closes immediately
- Task reference tracking: `background_tasks.add(task)` + `task.add_done_callback(background_tasks.discard)` prevents GC
- Guard checks: `admin_client is not None` ensures graceful degradation when Admin Agent registration failed
- `admin_item_id` truthiness check prevents triggering on empty string item IDs
- `raw_text` for text captures is `user_text` (the original capture); for voice captures, prefer `transcript_text`
- No `conversation_id` in the Admin Agent call (handled inside `process_admin_capture` which creates fresh messages)
