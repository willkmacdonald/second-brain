<interfaces>
GA streaming pattern (per probe 1):
```
from agent_framework import Agent, AgentSession

session = AgentSession(session_id=thread_id) if thread_id else AgentSession()
async for update in agent.run(question, session=session, stream=True):
    # update.role == 'assistant' always
    # update.text — accumulated user-visible text (empty during tool execution)
    # update.contents — list of Content objects (tool calls, tool results, text)
    # update.finish_reason — always None on streaming deltas
    text_delta = getattr(update, "text", "")
    if text_delta:
        yield encode_sse({"type": "text", "content": text_delta})
    for content in getattr(update, "contents", None) or []:
        # Existing content.type matching for function_call / function_result
        ...

# After stream completes:
session_id = session.session_id  # client-generated UUID
```

Existing SSE wire format (KEEP unchanged):
- `{"type": "text", "content": ...}` for text deltas
- `{"type": "tool_call", "tool": ..., "args": ...}` for function calls
- `{"type": "tool_result", "tool": ..., "result": ...}` for function results
- `{"type": "error", "message": ...}` for errors
- `{"type": "done", "thread_id": ...}` for completion

Custom span deletion (F-15):
The current code has `with tracer.start_as_current_span("investigate") as span:` wrapping the entire stream processing. Delete this `with` block. The framework's `invoke_agent` span (auto-emitted) is now the canonical agent span; capture-trace middleware tags it at source. The `investigate.question_length` and `investigate.thread_id` attributes either move:
- Onto `invoke_agent` via `trace.get_current_span().set_attribute(...)` IMMEDIATELY before `agent.run(...)` is called (the framework span IS current at that point), OR
- Onto AppRequests via the api endpoint (mirror `api/capture.py:228` pattern), which is the cleaner choice.

We pick the AppRequests path: the endpoint sets the attributes before calling into the adapter generator.
