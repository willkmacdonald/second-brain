## Decisions Made

- **Seed script field-name fix (auto-applied in Task 2):** Plan spec used `userId="eval-user-admin"` and `captureText`, but the existing admin eval runner is hardcoded to `WHERE c.userId = "will"` and reads `case["inputText"]`. The seed script pins to the runner's contract (userId="will", inputText) to ensure seeded cases are visible. Documented in script's module docstring + commit `8ac17d1` body.
- **Task captures use `expected_destination: "unrouted"`** because `DryRunAdminTools.add_task_items` captures no destination, leaving `captured_destinations` empty. The runner then predicts `"unrouted"` as a fallback. This is the correct expected value for non-routed admin captures.
- **Curation strategy: exclude known agent failures** (Home Depot destination creation, hardware affinity rule) — those are agent capability bugs, not migration regressions. They belong in a separate regression test, not the migration baseline.
- **Per-destination breakdown captured:** `jewel: 0.6667` (2/3 errand correct), `unrouted: 1.0` (8/8 task correct). The one misprediction was an errand case that fell through to `unrouted` instead of routing to `jewel`.
