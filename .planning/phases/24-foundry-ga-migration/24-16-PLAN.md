<interfaces>
GA streaming pattern (per probe 1):
```
from agent_framework import Agent, AgentSession, ChatOptions

session = AgentSession(session_id=stored_session_id) if stored_session_id else AgentSession()
options = ChatOptions(tool_choice="required")  # string form per probe 3

try:
    async with asyncio.timeout(60):
        async for update in agent.run(user_text, options=options, session=session, stream=True):
            for content in getattr(update, "contents", None) or []:
                if content.type == "function_call":
                    yield encode_sse({"type": "tool_call", ...})
                elif content.type == "function_result":
                    # file_capture result — extract bucket, confidence, status, item_id
                    parsed = _parse_result(content.result)
                    if parsed["status"] == "filed":
                        yield encode_sse({"type": "CLASSIFIED", ...})
                    elif parsed["status"] == "low_confidence":
                        yield encode_sse({"type": "LOW_CONFIDENCE", ...})
                    elif parsed["status"] == "misunderstood":
                        yield encode_sse({"type": "MISUNDERSTOOD", ...})
                    ...
            # text deltas
            text_delta = getattr(update, "text", "")
            if text_delta:
                yield encode_sse({"type": "text", "content": text_delta})

    # After stream completes:
    final_session_id = session.session_id
    yield encode_sse({"type": "COMPLETE", "sessionId": final_session_id})

except Exception as exc:
    # tool_choice='required' failed
    logger.error("Forced tool failure", exc_info=True, extra=...)
    yield encode_sse({
        "type": "ERROR",
        "sub_code": "forced_tool_failure",
        "message": "Classifier could not file this capture.",
    })
```

Wire-side compatibility:
- The 6 SSE event types (CLASSIFIED, LOW_CONFIDENCE, MISUNDERSTOOD, UNRESOLVED, ERROR, COMPLETE) preserved.
- The "thread_id" field on COMPLETE is renamed to "sessionId" — but mobile app may expect "thread_id". Investigate the mobile expectation. If mobile reads "thread_id", emit BOTH fields with the same value for backward compat:
  ```
  yield encode_sse({"type": "COMPLETE", "sessionId": session_id, "thread_id": session_id})
  ```
  Or rename Inbox.foundryThreadId to sessionId AND keep emitting thread_id on the wire. Mobile changes are out-of-scope for Phase 24 per CONTEXT.

PICK: emit BOTH sessionId AND thread_id on the SSE wire for backward compat. Mobile keeps reading thread_id; backend canonicalizes sessionId on Inbox.
