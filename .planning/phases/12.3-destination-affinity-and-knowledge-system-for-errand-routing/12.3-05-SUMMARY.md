## Decisions Made
- Horizontal scrollable chips (not dropdown/modal picker) for destination selection -- faster to tap, better for mobile UX
- Admin notification banners render above the processing banner -- notifications are user-facing responses that need attention
- Route requests always set saveRule: true -- the system auto-learns preferences by default, reducing future unrouted items
- "Other" chip always appended last in RoutingPicker with muted styling -- catch-all for destinations not yet created
