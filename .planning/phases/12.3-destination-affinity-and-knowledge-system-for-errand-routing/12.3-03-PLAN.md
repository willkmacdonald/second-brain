**Modify admin_handoff.py -- Part A: Routing context injection:**

Add a helper function `_build_routing_context()` that loads destinations and rules, then formats them as a context string prepended to the user message:

```python
async def _build_routing_context(cosmos_manager: CosmosManager) -> str:
    """Load destinations and affinity rules for Admin Agent context.

    Returns a formatted string to prepend to the user's capture text,
    giving the Admin Agent knowledge of available destinations and rules.
    """
    dest_container = cosmos_manager.get_container("Destinations")
    rules_container = cosmos_manager.get_container("AffinityRules")

    destinations: list[dict] = []
    async for doc in dest_container.query_items(
        query="SELECT * FROM c", partition_key="will"
    ):
        destinations.append(doc)

    rules: list[dict] = []
    async for doc in rules_container.query_items(
        query="SELECT * FROM c", partition_key="will"
    ):
        rules.append(doc)

    # Format context
    lines = ["AVAILABLE DESTINATIONS:"]
    for d in destinations:
        lines.append(f"- {d['slug']} ({d['displayName']}, {d.get('type', 'physical')})")

    if rules:
        lines.append("\nROUTING RULES:")
        for r in rules:
            lines.append(f"- {r['naturalLanguage']} ({r['ruleType']}: {r['itemPattern']} -> {r['destinationSlug']})")
            if r.get("exceptions"):
                for exc in r["exceptions"]:
                    lines.append(f"  EXCEPT: {exc['pattern']} -> {exc['destinationSlug']}")
    else:
        lines.append("\nNo routing rules defined yet. Route items using your best judgment of the available destinations.")

    lines.append("\nFor items with no clear destination match, set destination to 'unrouted'.")

    return "\n".join(lines)
```

**Modify admin_handoff.py -- Part B: Store Admin Agent response text for user delivery:**

This is critical for surfacing rule query answers, conflict confirmations, rule creation feedback, and destination deletion reassignment prompts to the user.

After the Admin Agent responds successfully (after the `response = await admin_client.get_response(...)` call), store the agent's response text on the inbox item BEFORE deleting it. Change the success flow:

1. Instead of immediately deleting the inbox item when tool_was_called is True, first update the inbox item with:
   ```python
   doc = await inbox_container.read_item(item=inbox_item_id, partition_key="will")
   doc["adminProcessingStatus"] = "completed"
   doc["adminAgentResponse"] = response.text if response.text else None
   await inbox_container.upsert_item(body=doc)
   ```
2. Then delete the inbox item ONLY if the response text is empty/None or is a simple confirmation (just errand items added). Use a heuristic: if `response.text` contains question marks, the words "conflict", "currently goes to", "Where should", "has N items", or "rule" -- keep the inbox item alive with status "completed" so the response can be surfaced.
3. Add a helper to classify whether a response needs user attention:
   ```python
   def _response_needs_delivery(response_text: str | None) -> bool:
       """Check if the Admin Agent response contains information the user needs to see.

       Returns True for rule queries, conflict confirmations, destination
       management feedback, etc. Returns False for simple errand-add confirmations.
       """
       if not response_text:
           return False
       indicators = ["?", "conflict", "currently goes to", "where should",
                      "rule", "created", "updated", "deleted", "destination",
                      "removed", "renamed"]
       text_lower = response_text.lower()
       return any(indicator in text_lower for indicator in indicators)
   ```
4. If `_response_needs_delivery()` returns True: set status to "completed" with `adminAgentResponse`, do NOT delete.
5. If `_response_needs_delivery()` returns False: delete as before (simple errand adds).
