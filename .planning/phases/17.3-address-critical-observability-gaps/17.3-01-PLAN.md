**Step 2: Create `mobile/lib/sentry.ts`** -- centralized Sentry init module.
```typescript
import * as Sentry from "@sentry/react-native";
import { isRunningInExpoGo } from "expo";

const navigationIntegration = Sentry.reactNavigationIntegration({
  enableTimeToInitialDisplay: !isRunningInExpoGo(),
});

export function initSentry(): void {
  Sentry.init({
    dsn: process.env.EXPO_PUBLIC_SENTRY_DSN ?? "",
    tracesSampleRate: 1.0,       // 100% for single-user app
    enableNativeFramesTracking: !isRunningInExpoGo(),
    integrations: [navigationIntegration],
    sendDefaultPii: false,       // No PII -- good hygiene even for single-user
    enabled: !__DEV__,           // Disabled in dev to avoid noise (Pitfall 4 from research)
  });
}

export { Sentry, navigationIntegration };
```

Key decisions:
- `enabled: !__DEV__` prevents dev-mode double-counting (React rethrows boundary errors in dev)
- `tracesSampleRate: 1.0` because this is a single-user app (no cost concern)
- `sendDefaultPii: false` for good hygiene
- `isRunningInExpoGo()` guards native-only features

**Step 2: Rewrite `mobile/app/_layout.tsx`** to integrate Sentry.

CRITICAL: `initSentry()` must be called at MODULE SCOPE (outside the component function)
so it runs before any rendering. If it's inside useEffect, early crashes are missed (Pitfall 3).

The modified layout must:
1. Import and call `initSentry()` at module scope (top of file, after imports)
2. Import `Sentry` and `navigationIntegration` from `../lib/sentry`
3. Import `ErrorFallback` from `../components/ErrorFallback`
4. Import `useNavigationContainerRef` from `expo-router`
5. Import `useEffect` from `react`
6. Inside the component, get `ref = useNavigationContainerRef()` and register it with
   `navigationIntegration.registerNavigationContainer(ref)` in a useEffect
7. Wrap the Stack inside `<Sentry.ErrorBoundary fallback={ErrorFallback}>`
8. Export the component wrapped with `Sentry.wrap(RootLayout)` as the default export

The final file should look like:
```typescript
import { useEffect } from "react";
import { Stack, useNavigationContainerRef } from "expo-router";
import { GestureHandlerRootView } from "react-native-gesture-handler";
import * as Sentry from "@sentry/react-native";
import { initSentry, navigationIntegration } from "../lib/sentry";
import { ErrorFallback } from "../components/ErrorFallback";

// Initialize Sentry at module scope -- BEFORE any rendering (Pitfall 3)
initSentry();

function RootLayout() {
  const ref = useNavigationContainerRef();

  useEffect(() => {
    if (ref?.current) {
      navigationIntegration.registerNavigationContainer(ref);
    }
  }, [ref]);

  return (
    <GestureHandlerRootView style={{ flex: 1 }}>
      <Sentry.ErrorBoundary fallback={ErrorFallback}>
        <Stack screenOptions={{ headerShown: false }}>
          <Stack.Screen name="(tabs)" />
          <Stack.Screen
            name="conversation/[threadId]"
            options={{
              headerShown: true,
              headerTitle: "",
              headerStyle: { backgroundColor: "#0f0f23" },
              headerTintColor: "#ffffff",
            }}
          />
        </Stack>
      </Sentry.ErrorBoundary>
    </GestureHandlerRootView>
  );
}

export default Sentry.wrap(RootLayout);
```

IMPORTANT: Preserve the exact same Stack.Screen configuration (tab layout + conversation screen
with header options). The only structural changes are: Sentry.ErrorBoundary wrapping the Stack,
Sentry.wrap() on the export, and the navigation ref/useEffect hook.

DO NOT modify `mobile/lib/telemetry.ts` -- the existing reportError() pipeline is complementary
to Sentry, not replaced by it (operational events vs crash reporting).
