## Decisions Made
- Sentry disabled in __DEV__ mode to prevent dev-mode double-counting (React rethrows boundary errors in dev)
- tracesSampleRate set to 1.0 -- appropriate for single-user app with no cost pressure
- sendDefaultPii set to false for good hygiene even though this is a personal app
- isRunningInExpoGo() guards native-only features (native frames tracking, time-to-initial-display)
- Used placeholder values for org slug, project slug, and DSN -- user replaces before first EAS build

## User Setup Required

**External services require manual configuration before first build:**

1. Create a Sentry account at https://sentry.io/signup/ (free Developer tier: 5,000 errors/month)
2. Create a React Native project in Sentry
3. Replace placeholders in the codebase:
   - `mobile/app.json`: Replace `"SENTRY_ORG_SLUG"` and `"SENTRY_PROJECT_SLUG"` with actual values
   - `mobile/eas.json`: Replace `"SENTRY_DSN_PLACEHOLDER"` with actual DSN in both build profiles
4. Create auth token and set EAS secret: `eas secret:create --name SENTRY_AUTH_TOKEN --value <token>`
5. Run `eas build --profile development --platform ios` (native module requires new dev build)
