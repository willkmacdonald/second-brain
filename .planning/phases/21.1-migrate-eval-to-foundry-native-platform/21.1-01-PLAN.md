<objective>
Replace the custom eval framework with Azure AI Foundry's native evaluation platform.

Purpose: The project's guiding principle is to leverage Foundry/Azure built-in capabilities before building custom. Phase 21 incorrectly concluded Foundry lacked relevant evaluators. This plan replaces custom scoring/result storage with Foundry's native evaluation APIs via AIProjectClient. Direct Foundry agent-target evaluation is preferred, but it is gated by a canary because the current app supplies Python tools at request time. If the canary cannot prove production-equivalent tool capture, the backend generates response/tool-call artifacts through the production app clients/tools and Foundry still performs evaluator scoring, run tracking, and result storage. Two custom code-based evaluators (classifier bucket accuracy, admin routing accuracy) are registered in the Foundry evaluator catalog, and built-in evaluators (IntentResolution, ToolCallAccuracy, TaskAdherence) run alongside them. Investigation tools become the sole trigger for eval runs per D-05/D-06.

Output: eval/foundry.py with Foundry SDK functions, canary-gated execution-mode selection, rewired investigation tools, cleaned main.py, deleted old eval code after replacement exists, new tests.
</objective>

**Constants:**
- `CLASSIFIER_GRADE_FN`: String containing the `grade(sample, item) -> float` function for classifier bucket accuracy. This function runs in Foundry's sandbox with NO network. It must support both execution modes:
  - Direct target mode: parse `item.get("sample", {}).get("output_items")` to find `file_capture` tool calls and extract the `bucket` argument.
  - App-mediated fallback mode: parse `item.get("tool_calls")` from precomputed JSONL artifacts.
  Compare predicted bucket (lowercased, stripped) to `item.get("expected_bucket")`. Return 1.0 on exact match, 0.0 otherwise. Wrap parsing defensively and return 0.0 on malformed data rather than raising.
- `ADMIN_GRADE_FN`: String containing the `grade(sample, item) -> float` function for admin routing accuracy. It must support both execution modes:
  - Direct target mode: parse `item.get("sample", {}).get("output_items")`.
  - App-mediated fallback mode: parse `item.get("tool_calls")`.
  Find `add_errand_items` or `add_task_items`; for errands compare destination slug(s) to `item.get("expected_destination")`, and for task calls treat the destination as `"tasks"`. Return 1.0 on exact destination match, 0.0 otherwise.

**Functions:**

a) `async def ensure_evaluators_registered(project_client: AIProjectClient) -> dict[str, str]`:
   - Idempotently ensure two custom code-based evaluators exist using `project_client.beta.evaluators.*` (per D-01, D-03, D-15).
   - Compute a stable code hash for each evaluator string. Reuse the latest existing evaluator version if its metadata/description includes the same hash; create a new version if the hash differs or the evaluator does not exist.
   - Evaluator 1: name="classifier_bucket_accuracy", category=EvaluatorCategory.QUALITY, type=EvaluatorDefinitionType.CODE, code_text=CLASSIFIER_GRADE_FN, init_parameters with deployment_name (string, required) and pass_threshold (number, required), metrics with "result" (continuous, 0-1, increase), data_schema requiring item.expected_bucket.
   - Evaluator 2: name="admin_routing_accuracy", same structure but with ADMIN_GRADE_FN and data_schema requiring item.expected_destination.
   - Return dict mapping evaluator name to evaluator/version ID.
   - Wrap in try/except, log errors with component="eval".

b) `async def export_and_upload_dataset(project_client: AIProjectClient, cosmos_manager: CosmosManager, eval_type: str) -> dict`:
   - Query GoldenDataset container for cases matching eval_type ("classifier" = all entries, "admin_agent" = entries with expectedDestination not None).
   - Write to a tempfile as JSONL. For classifier direct mode: `{"query": inputText, "expected_bucket": expectedBucket}`. For admin direct mode: include deterministic routing context in the query text, e.g. `{"query": f"{routing_context}\n\n---\nUser capture: {inputText}", "expected_destination": expectedDestination}` unless the canary proves the target agent can call `get_routing_context`.
   - Compute a content hash from normalized JSONL rows.
   - Upload via `project_client.datasets.upload_file(name=f"{eval_type}-golden-dataset", version=f"{timestamp}-{hash8}", file_path=jsonl_path)`.
   - Return a dict containing `file_id`, `dataset_hash`, `dataset_version`, and row count.
   - Use `asyncio.to_thread()` for the sync `upload_file` call.

c) `async def run_foundry_target_canary(project_client: AIProjectClient, cosmos_manager: CosmosManager) -> dict`:
   - Run a minimal classifier canary row and admin canary row through direct `azure_ai_target_completions` against `Classifier` and `AdminAgent`.
   - Assert classifier output items contain a `file_capture` tool call with a bucket argument.
   - Assert admin output items contain `add_errand_items` or `add_task_items`, or prove `get_routing_context` can be called by target eval.
   - Return `{"direct_target_supported": bool, "classifier_tool_capture": bool, "admin_tool_capture": bool, "details": ...}`.
   - This is a blocking gate before deleting the old app-mediated eval invocation path.

d) `async def generate_app_mediated_dataset(... ) -> dict`:
   - Fallback-only helper. Use the existing production classifier/admin clients and tool lists to run golden dataset rows through the same app-mediated path used in production.
   - Capture `response_text` and normalized `tool_calls` into JSONL rows.
   - For admin rows, prepend the deterministic `build_routing_context()` output exactly as production does before invoking the AdminAgent.
   - Upload artifact JSONL to Foundry with content-hash/timestamp versioning and return `file_id`, `dataset_hash`, `dataset_version`, and row count.

e) `async def run_classifier_eval(project_client: AIProjectClient, cosmos_manager: CosmosManager, *, classifier_client=None, classifier_tools=None, execution_mode: str | None = None) -> dict`:
   - Get openai_client via `project_client.get_openai_client()`.
   - Call `ensure_evaluators_registered(project_client)` before creating eval definitions.
   - Determine execution mode: use direct target only if `execution_mode == "direct_target"` or cached canary status says direct target is supported; otherwise use app-mediated artifact fallback.
   - Upload direct-mode dataset via `export_and_upload_dataset(...)`, or fallback artifact dataset via `generate_app_mediated_dataset(...)`.
   - Create eval definition with DataSourceConfigCustom (type="custom", item_schema with query + expected_bucket, include_sample_schema=True).
   - Testing criteria: (1) custom "classifier_bucket_accuracy" evaluator with init_parameters {deployment_name: "gpt-4o", pass_threshold: 0.5}, (2) builtin "intent_resolution" evaluator with init_parameters {deployment_name: "gpt-4o"} and data_mapping {query: "{{item.query}}", response: "{{sample.output_items}}"} (per D-02).
   - Direct mode: create eval run with data_source type="azure_ai_target_completions", source from uploaded file_id, input_messages template with {{item.query}}, target type="azure_ai_agent" name="Classifier".
   - Fallback mode: create eval run with data_source type="jsonl" over artifact rows and map evaluator inputs from `item.tool_calls` / `item.response_text`.
   - Use stable eval name `second-brain-classifier-eval` and run name prefix `classifier-YYYYMMDD-HHMMSS-<dataset_hash>`.
   - All sync SDK calls wrapped in `asyncio.to_thread()`.
   - Return dict with eval_id, run_id, status, execution_mode, dataset_hash, dataset_version.

f) `async def run_admin_eval(project_client: AIProjectClient, cosmos_manager: CosmosManager, *, admin_client=None, admin_tools=None, execution_mode: str | None = None) -> dict`:
   - Same pattern as classifier but: dataset has query + expected_destination, target name="AdminAgent", testing criteria include custom "admin_routing_accuracy" + builtin "tool_call_accuracy" + builtin "task_adherence" (per D-02).
   - Preserve deterministic routing context in both direct and fallback modes unless the canary proves target eval can call `get_routing_context`.
   - Use stable eval name `second-brain-admin-eval` and run name prefix `admin-YYYYMMDD-HHMMSS-<dataset_hash>`.

g) `async def poll_eval_run(project_client: AIProjectClient, eval_id: str, run_id: str) -> dict`:
   - Poll `openai_client.evals.runs.retrieve(run_id=run_id, eval_id=eval_id)` every 5 seconds until status is "completed" or "failed".
   - Wrap sync calls in `asyncio.to_thread()`.
   - Return run dict with status, report_url.

h) `async def get_eval_results_from_foundry(project_client: AIProjectClient, eval_id: str, run_id: str) -> dict`:
   - Retrieve run via `openai_client.evals.runs.retrieve()`.
   - List output items via `openai_client.evals.runs.output_items.list()`.
   - Parse results: count total, count passed, compute accuracy, build per-category breakdown, list failures with input text + expected vs actual.
   - Format output to match existing investigation tool contract (per D-11): `{"accuracy": float, "total": int, "correct": int, "per_bucket": {...}, "failures": [...]}`.
   - All sync SDK calls wrapped in `asyncio.to_thread()`.

i) `async def list_recent_eval_runs(project_client: AIProjectClient, eval_type: str | None = None, limit: int = 3) -> list[dict]`:
   - List recent eval objects and their latest runs using stable eval names (`second-brain-classifier-eval`, `second-brain-admin-eval`) and run prefixes.
   - Return list of {eval_id, run_id, name, status, created_at, report_url}.
   - Wrap sync calls in `asyncio.to_thread()`.

Use `logger = logging.getLogger(__name__)` throughout. All functions use TYPE_CHECKING guard for CosmosManager import. Follow the logging pattern from PATTERNS.md (component="eval", eval_type, eval_run_id in extras).
