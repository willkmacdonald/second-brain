<objective>
Replace the custom eval framework with Azure AI Foundry's native evaluation platform.

Purpose: The project's guiding principle is to leverage Foundry/Azure built-in capabilities before building custom. Phase 21 incorrectly concluded Foundry lacked relevant evaluators. This plan deletes the custom eval pipeline (runner, dry-run tools, metrics, API endpoints, 7 files) and replaces it with Foundry's native agent target evaluation via AIProjectClient. Two custom code-based evaluators (classifier bucket accuracy, admin routing accuracy) are registered in the Foundry evaluator catalog, and built-in evaluators (IntentResolution, ToolCallAccuracy, TaskAdherence) run alongside them. Investigation tools become the sole trigger for eval runs per D-05/D-06.

Output: eval/foundry.py with Foundry SDK functions, rewired investigation tools, cleaned main.py, deleted old eval code, new tests.
</objective>

It parses `item.get("sample", {}).get("output_items")` to find `file_capture` tool calls and extract the `bucket` argument. Compares predicted bucket (lowercased, stripped) to `item.get("expected_bucket")` (lowercased, stripped). Returns 1.0 on match, 0.0 otherwise. - `ADMIN_GRADE_FN`: String containing the `grade(sample, item) -> float` function for admin routing accuracy. Parses tool calls to find `add_errand_items` or `add_task_items` and extracts the destination argument. Compares to `item.get("expected_destination")` (lowercased, stripped). Returns 1.0 on match, 0.0 otherwise.

**Functions:**

a) `async def register_evaluators(project_client: AIProjectClient) -> dict[str, str]`:

   async def _poll_and_update(
       project_client: AIProjectClient,
       eval_id: str,
       run_id: str,
       runs_dict: dict,
   ) -> None:
       """Background task: poll Foundry eval run until completion."""
       try:

           runs_dict[eval_id]["status"] = result.get("status", "unknown")
