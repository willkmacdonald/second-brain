## Decisions Made

- AIProjectClient uses sync DefaultAzureCredential because it wraps a sync OpenAI client; all eval SDK calls wrapped in asyncio.to_thread() to avoid blocking the event loop
- Foundry project client initialization is non-fatal (warning + None) because eval features are not on the critical capture path
- Custom evaluator grade() functions defensively parse both direct target output_items and app-mediated tool_calls JSONL for dual-mode support
- Evaluator registration uses code-hash matching in the description field for idempotent create/reuse behavior
- Dataset uploads use `{timestamp}-{content_hash[:8]}` versioning to prevent stale data and version conflicts
- Kept classifier_client and admin_client parameters on InvestigationTools for app-mediated fallback until direct target canary proves they are unnecessary
- Agent-framework-azure-ai left unpinned in pyproject.toml to avoid lockfile resolution conflicts with azure-ai-projects stable 2.1.0
