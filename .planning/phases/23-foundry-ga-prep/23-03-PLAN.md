App Insights span query template (per MEMORY.md):
```kql
union withsource=SourceTable AppRequests, AppDependencies, AppTraces, AppExceptions
| where TimeGenerated > ago(2h)
| where tostring(Properties.["capture.trace_id"]) == "<TRACE_ID>"
   or tostring(Properties.capture_trace_id) == "<TRACE_ID>"
| extend ItemType = case(
    SourceTable == "AppRequests", "Request",
    SourceTable == "AppDependencies", "Dependency",
    SourceTable == "AppTraces", "Trace",
    SourceTable == "AppExceptions", "Exception",
    "Unknown"
  )
| project TimeGenerated, ItemType, Name, ResultCode, DurationMs, OperationId, ParentId, Id, Properties
| order by TimeGenerated asc
```

This pattern matches the canonical `union withsource=SourceTable ... | extend ItemType = case(...)` form used in `backend/src/second_brain/observability/kql_templates.py:44, 74, 220`. The `type()` function does NOT exist in KQL (it returns column type, not table source); `$table` is not a valid identifier in this context. Use the SourceTable + case() pattern.
