    - `POST /api/eval/run` request body:
      - `eval_type` MUST be either the literal string `"classifier"` or `"admin_agent"`. **NOT `"admin"`** — the deployed handler rejects `"admin"` with HTTP 400.
      - `routing_context` is REQUIRED when `eval_type == "admin_agent"` (per `eval.py:55-58`); MUST be omitted (or null) for classifier.
    - `POST /api/eval/run` response shape:
      - `runId` (camelCase — NOT `run_id`)
      - `status` (e.g. `"running"`)
      - `evalType` (camelCase — echoes the request)
    - Polling: `GET /api/eval/status/{run_id}` (path is `/status/`, NOT `/run/`).
    - **CRITICAL: the status endpoint at `/api/eval/status/{run_id}` does NOT include `evalType` in its final response** — the runner overwrites the runs dict at `runner.py:205-211` (classifier) and `runner.py:355-365` (admin) with `{"status": "completed", "result_id": ..., "accuracy"|"routing_accuracy", "total", "correct"}` and does NOT preserve the `evalType` field. The status endpoint then returns `{"runId": run_id, **run}` (eval.py:124), so the final status response also lacks `evalType`.
