<decisions>
## Implementation Decisions

### Observability audit
- Audit ALL telemetry paths first: what's visible where, what gaps exist between App Insights, AI Foundry, and Sentry
- Check existing docs/instrumentation.md for what's already documented
- Goal is a single pane of glass in App Insights — no switching between portals to understand what happened
- App Insights should surface Foundry agent internals (thinking time, tool routing, model inference), not just our-side OTel spans

### Agent diagnostics scope
- All three agents (Classifier, Admin Agent, Investigation Agent) need Foundry-side visibility
- Both debugging (why is this call slow?) and ongoing monitoring (health metrics, latency percentiles)
- Metrics land in App Insights, not Foundry portal

### Alerting
- Add new alert rules to existing SecondBrainAlerts action group (push notifications to phone)
- Alert on: individual timeouts (>60s), slow thinking time (>30s before tool call), consecutive failures (3+ in 10 min)
- Alert only, manual restart — no auto-recovery

### Container App resilience
- Enhanced /health endpoint that verifies Foundry agent connection is alive; reports degraded if stale
- 5-minute warmup should self-heal: if agent is unresponsive, recreate the AzureAIAgentClient
- Cut /api/investigate timeout from 60s to 30s with user-friendly error message

### Codex code review findings
- Fix ALL findings from codex-code-review.md:
  - **High #1:** Sentry ErrorFallback prop type mismatch (error: unknown vs Error)
  - **High #2:** Recipe URL extraction DNS test failures (6 failed tests, SSRF check coupled to live DNS)
  - **Minor #3:** Stale planning docs still say "Last Error" instead of "Errors (24h)"
  - **Minor #4:** Errands API tests leak unawaited coroutine from mocked asyncio.create_task
  - **Open Q:** MOBL-04/MOBL-05 requirements mention eval scores that were deferred — reconcile docs

### Codex CI integration
- Add Codex code review as a CI step (GitHub Actions) so every push gets automated review

### Claude's Discretion
- How to wire Foundry traces into App Insights (SDK instrumentation, custom spans, or API polling)
- Specific KQL queries for agent health dashboard
- Alert rule thresholds can be tuned after seeing baseline data
- Codex CI configuration details (trigger on push, PR, or both)

</decisions>
