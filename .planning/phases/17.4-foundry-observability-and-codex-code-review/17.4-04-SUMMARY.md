## Decisions Made
- Used `az rest` with Scheduled Query Rules REST API instead of `az monitor scheduled-query create` (CLI had scope resolution issues)
- Alert KQL uses portal-style table names (`dependencies`/`traces`) not workspace-style (`AppDependencies`/`AppTraces`) since rules scope to App Insights resource directly
- Custom middleware spans (`classifier_agent_run` etc.) nest as child spans under SDK's `process_thread_run` — agent differentiation works via `gen_ai.agent.id` attribute on SDK spans

## Deviations from Plan
- Alert KQL table names adapted from workspace-style to portal-style (known schema difference documented in project memory)
- `az monitor scheduled-query create` CLI failed; fell back to `az rest` as the plan suggested
