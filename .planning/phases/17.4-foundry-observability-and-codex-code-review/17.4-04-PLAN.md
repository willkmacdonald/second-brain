<task type="auto">
  <name>Task 1: Create Azure Monitor alert rules for agent health</name>
  <files></files>
  <action>
  Create 3 Azure Monitor log search alert rules using `az monitor scheduled-query create`. All rules target the `second-brain-insights` App Insights resource in `shared-services-rg` and use the `SecondBrainAlerts` action group.

  **Alert 1: Agent Timeout (>60s)**
  ```bash
  az monitor scheduled-query create \
    --name "Agent-Timeout" \
    --resource-group shared-services-rg \
    --scopes "/subscriptions/{sub_id}/resourceGroups/shared-services-rg/providers/Microsoft.Insights/components/second-brain-insights" \
    --condition "count > 0" \
    --condition-query "AppDependencies | where TimeGenerated > ago(5m) | where Name startswith 'invoke_agent' or Name has 'investigate' or Name has 'classifier' or Name has 'admin' | where DurationMs > 60000" \
    --window-size 5m \
    --evaluation-frequency 5m \
    --severity 2 \
    --action-groups "/subscriptions/{sub_id}/resourceGroups/shared-services-rg/providers/Microsoft.Insights/actionGroups/SecondBrainAlerts" \
    --description "An agent call took longer than 60 seconds"
  ```

  **Alert 2: Slow Agent Thinking (>30s before tool call)**
  ```bash
  az monitor scheduled-query create \
    --name "Agent-Slow-Thinking" \
    --resource-group shared-services-rg \
    --scopes "/subscriptions/{sub_id}/resourceGroups/shared-services-rg/providers/Microsoft.Insights/components/second-brain-insights" \
    --condition "count > 0" \
    --condition-query "AppDependencies | where TimeGenerated > ago(5m) | where Name startswith 'chat' | where DurationMs > 30000" \
    --window-size 5m \
    --evaluation-frequency 5m \
    --severity 3 \
    --action-groups "/subscriptions/{sub_id}/resourceGroups/shared-services-rg/providers/Microsoft.Insights/actionGroups/SecondBrainAlerts" \
    --description "LLM inference took longer than 30 seconds (agent thinking time)"
  ```

  **Alert 3: Consecutive Agent Failures (3+ in 10min)**
  ```bash
  az monitor scheduled-query create \
    --name "Agent-Consecutive-Failures" \
    --resource-group shared-services-rg \
    --scopes "/subscriptions/{sub_id}/resourceGroups/shared-services-rg/providers/Microsoft.Insights/components/second-brain-insights" \
    --condition "count > 0" \
    --condition-query "AppTraces | where TimeGenerated > ago(10m) | where SeverityLevel >= 3 | where Message has 'stream error' or Message has 'timed out' or Message has 'failed' or Message has 'Warmup ping failed' | where Properties has 'investigation' or Properties has 'classifier' or Properties has 'admin' | summarize ErrorCount = count() by bin(TimeGenerated, 10m) | where ErrorCount >= 3" \
    --window-size 10m \
    --evaluation-frequency 5m \
    --severity 2 \
    --action-groups "/subscriptions/{sub_id}/resourceGroups/shared-services-rg/providers/Microsoft.Insights/actionGroups/SecondBrainAlerts" \
    --description "3 or more agent errors in a 10-minute window"
  ```

  IMPORTANT: Before running these commands, first look up the subscription ID and the full resource ID of the App Insights resource and the action group:
  ```bash
  az account show --query id -o tsv
  az monitor app-insights component show --app second-brain-insights -g shared-services-rg --query id -o tsv
  az monitor action-group show --name SecondBrainAlerts -g shared-services-rg --query id -o tsv
  ```

  If `az monitor scheduled-query create` syntax differs from the above, check `az monitor scheduled-query create --help` and adapt. The exact CLI flags may vary -- the key parameters are the KQL query, window size, evaluation frequency, severity, and action group.

  If the az CLI approach is too complex or the flags don't match, create the alerts via `az rest` calling the Azure Monitor REST API directly, or document the exact KQL queries and alert configuration for manual portal setup.

  ```markdown
  ## Agent Framework GenAI Spans

  The Agent Framework SDK emits OpenTelemetry spans for every agent interaction when `enable_instrumentation()` is called (in `main.py` after `configure_azure_monitor()`):

  | Span Name | What It Tracks | Where in App Insights |
  |-----------|---------------|----------------------|
  | `invoke_agent <agent_name>` | Full agent run lifecycle | AppDependencies |
  | `chat <model_name>` | LLM inference call (thinking time) | AppDependencies |
  | `execute_tool <function_name>` | Tool execution within agent run | AppDependencies |
  | `{agent_name}_agent_run` | Custom middleware span (classifier/admin/investigation) | AppDependencies (via OTel tracer) |
  | `tool_{function_name}` | Custom tool timing middleware | AppDependencies (via OTel tracer) |

  **Metrics automatically tracked:**
  - `gen_ai.usage.input_tokens` -- input tokens per call
  - `gen_ai.usage.output_tokens` -- output tokens per call
  - `gen_ai.client.operation.duration` -- operation duration histogram

  **Content recording:**
  Set `ENABLE_SENSITIVE_DATA=true` as a Container App env var to include prompt/response content in spans. This is appropriate for this single-user system and provides enormous debugging value.
  ```
