# Phase 20.1: Design Team UI Improvements - Context

**Gathered:** 2026-04-22
**Status:** Ready for planning

<domain>
## Phase Boundary

Apply the design team's hi-fi visual overhaul to the existing React Native/Expo mobile app. This is a visual reskin driven by the `docs/ui-design/` handoff bundle — new color tokens, typography system, refined screen layouts, and polished interaction details. All existing screens get the treatment. No new backend capabilities or features are added.

</domain>

<decisions>
## Implementation Decisions

### Scope & rollout strategy
- **D-01:** Foundation first — build design tokens + theme system + typography before reskinning any screens. Each screen reskin builds on the foundation.
- **D-02:** All existing screens in scope: Capture (3 states), Inbox (list + detail modal), Tasks (errands list), Status (dashboard + spine), Settings, and HITL Clarification flow.
- **D-03:** Daily Digest screen is NOT in scope — it's a new feature, not a reskin of something that exists.
- **D-04:** HITL Clarification flow gets the design treatment (bubble + 2×2 bucket grid style).

### Design token system
- **D-05:** Constants file (`theme.ts`) + `useTheme()` hook for access. No external styling library deps. `StyleSheet.create` stays as the styling mechanism.
- **D-06:** All three custom fonts: Instrument Serif (display headlines, italic), Instrument Sans (body/labels/buttons), JetBrains Mono (metadata/timestamps/caps labels). Install via `expo-font` + Google Fonts packages (`@expo-google-fonts/*`).
- **D-07:** Adopt design's bucket colors exactly — replace current palette with the design's muted variants: People=warm sand, Projects=dusty blue, Ideas=muted violet, Admin=sage. Each bucket has fg/bg/dot triple.
- **D-08:** Color tokens from `docs/ui-design/screens/tokens.jsx` are canonical — surfaces (bg, surface, surfaceHi, hairline, divider), text (text, textDim, textMuted, textFaint), accent, status colors (recording, ok, warn, err).

### Screen fidelity targets
- **D-09:** Capture screen: match all 3 design states (Idle with big mic + italic headline, Listening with waveform + live transcript, Filed with bucket confirmation). These states already exist — this is a visual upgrade.
- **D-10:** Status screen: keep real data (current dashboard cards + spine tiles), apply the design's card anatomy (surface bg, traffic dot, role line, status line). Don't add fake agents that don't exist.
- **D-11:** Tasks screen: restyle the existing errands/tasks list with the design's visual treatment (typography, colors, card style). No segmented toggle, no destination grouping, no GPS proximity — those are new features.
- **D-12:** Inbox screen: full reskin including the detail/recategorize modal — floating card, bucket grid, confidence display, thumbs up/down buttons. All underlying data exists from Phases 12 and 20.
- **D-13:** Settings screen: apply the design's grouped iOS-style list treatment to existing real settings (API key management, any functional toggles). Don't add rows for features that don't exist (digest time, eval schedule, entity resolution).

### Icons & tab bar
- **D-14:** Use Lucide icons (cross-platform, design README recommended). Match icon concepts from design (mic, inbox, checkbox, spark) with the design's active/inactive styling (stroke weight, color).
- **D-15:** Tab bar: 4 tabs (Capture | Inbox | Tasks | Status) — matches current structure from Phase 19.4.1. Apply design's visual treatment (dark translucent bg, backdrop blur, hairline top border, active/inactive states).

### Claude's Discretion
- Exact spacing values within the design's 2/4 multiple system
- Animation timing for capture flow transitions (design says ~200ms fade)
- Loading skeleton and error state implementations
- How to handle the waveform animation in Listening state (design uses IconWave helper)
- CapsLabel component implementation details
- Whether to use `expo-blur` for tab bar backdrop blur or a simpler opacity approach

</decisions>

<specifics>
## Specific Ideas

- Design README: "Typography is the brand." Instrument Serif italic headlines are load-bearing emotional moments — don't compromise on this.
- Design README: "The mic is sacred." The 86×86 mic button on idle is the emotional center. Give it room.
- Design README: "Buckets are a core concept." Always surface with color + icon + name, never just one.
- Design README: "This is a dark-first product." No light theme exists or is planned.
- Hairlines are 0.5px solid rgba(255,255,255,0.06) — always prefer hairlines over 1px full opacity.
- Screen content area: 320×660 logical px inside device frame. Safe areas: 42px top, 70px bottom tab bar.

</specifics>

<deferred>
## Deferred Ideas

- **Daily Digest / Morning Briefing** — Full-bleed daily summary with push notification delivery. New feature requiring backend endpoint + notification infrastructure.
- **GPS proximity notifications** — Tasks 'Where' view with store distances and geofencing. Needs location services + destination data model.
- **Agent Proposals (Tasks view C)** — Agent-suggested tasks with Accept/Skip flow. Needs new agent capability + proposal data model.
- **Settings detail rows for non-existent features** — Digest time, eval schedule, entity resolution schedule. These reference agents/features built in future phases.
- **Tasks segmented toggle (Where/Today/Proposals)** — Three-view toggle with destination grouping and priority ordering. New feature work, not a reskin.
- **Light theme** — Design is dark-only. Light theme would be a separate design + implementation effort.

</deferred>
