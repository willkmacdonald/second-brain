  <action>
Modify `mobile/app/_layout.tsx` to load all three custom font families before the app renders. Use the `useFonts` hook from `expo-font` gated with `expo-router`'s `SplashScreen`.

Add these imports at the top:
```typescript
import { useFonts } from 'expo-font';
import * as SplashScreen from 'expo-splash-screen';
import {
  InstrumentSerif_400Regular,
  InstrumentSerif_400Regular_Italic,
} from '@expo-google-fonts/instrument-serif';
import {
  InstrumentSans_400Regular,
  InstrumentSans_500Medium,
  InstrumentSans_600SemiBold,
  InstrumentSans_700Bold,
} from '@expo-google-fonts/instrument-sans';
import {
  JetBrainsMono_400Regular,
} from '@expo-google-fonts/jetbrains-mono';
```

Add `SplashScreen.preventAutoHideAsync();` BEFORE the RootLayout function definition (at module scope, after initSentry()).

Inside `RootLayout`, add the useFonts hook:
```typescript
const [fontsLoaded] = useFonts({
  InstrumentSerif_400Regular,
  InstrumentSerif_400Regular_Italic,
  InstrumentSans_400Regular,
  InstrumentSans_500Medium,
  InstrumentSans_600SemiBold,
  InstrumentSans_700Bold,
  JetBrainsMono_400Regular,
});
```

Add a useEffect that hides splash when fonts are loaded:
```typescript
useEffect(() => {
  if (fontsLoaded) {
    SplashScreen.hideAsync();
  }
}, [fontsLoaded]);
```

Add an early return before the existing JSX:
```typescript
if (!fontsLoaded) return null;
```

Also update the hardcoded header colors in Stack.Screen options to use theme tokens:
- Change all `headerStyle: { backgroundColor: "#0f0f23" }` to `headerStyle: { backgroundColor: '#0a0a12' }`
- Change all `headerTintColor: "#ffffff"` to `headerTintColor: '#f0f0f5'`

Do NOT restructure existing providers, error boundary, or Sentry wrapping. The font gate goes INSIDE RootLayout, before the return statement, not outside it.
