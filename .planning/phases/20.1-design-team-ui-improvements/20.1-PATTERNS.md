### `mobile/hooks/useTheme.ts` (NEW -- hook, context read)

**Analog:** `mobile/contexts/ApiKeyContext.tsx` (lines 1-57)

**Context + hook pattern** from `mobile/contexts/ApiKeyContext.tsx` (lines 16-56):
```typescript
const ApiKeyContext = createContext<ApiKeyContextType>({
  apiKey: "",
  setApiKey: async () => {},
  isLoading: true,
});

export function ApiKeyProvider({ children }: { children: ReactNode }) {
  // ...
  return (
    <ApiKeyContext.Provider value={{ apiKey, setApiKey, isLoading }}>
      {children}
    </ApiKeyContext.Provider>
  );
}

export const useApiKey = () => useContext(ApiKeyContext);
```

**Implementation notes:** Given that D-05 specifies a simple constants file + hook (NOT a context provider), `useTheme()` should simply re-export the theme constants from `theme.ts`. This avoids unnecessary context overhead for a static, non-reactive token system. A simple `export function useTheme() { return theme; }` wrapper is sufficient and can be upgraded to a context later if dynamic theming (light mode) ever becomes needed.

## Shared Patterns

### Color Token Replacement Map

**Apply to:** ALL modified files

Every file uses the same hardcoded color palette. The mapping is:

| Current Hardcoded | Theme Token | Description |
|-------------------|-------------|-------------|
| `"#0f0f23"` | `theme.colors.bg` | Background (deepened to `#0a0a12`) |
| `"#1a1a2e"` | `theme.colors.surface` | Card/surface (`#13131f`) |
| `"#16162b"` | `theme.colors.surface` | Alt surface (consolidate) |
| `"#1a2028"` | `theme.colors.surface` | Another alt surface |
| `"#2a2a4e"` | `theme.colors.surfaceHi` | Elevated surface (`#1b1b2a`) |
| `"#4a4a6e"` | `theme.colors.surfaceHi` | Alt elevated |
| `"#ffffff"` | `theme.colors.text` | Primary text (`#f0f0f5`) |
| `"#ccc"` / `"#cccccc"` | `theme.colors.textDim` | Secondary text (`rgba(240,240,245,0.62)`) |
| `"#999"` / `"#999999"` | `theme.colors.textDim` | Alt secondary |
