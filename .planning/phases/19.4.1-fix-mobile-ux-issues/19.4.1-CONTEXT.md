# Phase 19.4.1: Fix Mobile UX Issues - Context

**Gathered:** 2026-04-21
**Status:** Ready for planning

<domain>
## Phase Boundary

Restructure mobile tab navigation to separate actionable items (errands/tasks) from system observability (dashboard/spine), fix the stale error deep-link, add first-launch API key prompt with settings screen, and ship a production EAS build with TestFlight + OTA updates so the app works as a real standalone app away from home.

</domain>

<decisions>
## Implementation Decisions

### Tab Structure
- **D-01:** 4 tabs: Capture | Inbox | Tasks | Status. Tasks is a new tab; Status becomes purely observational.
- **D-02:** Tasks tab title is "Tasks" with ✅ icon. Status tab icon changes from ⚡ to 📊.
- **D-03:** Tasks tab shows errands + tasks only (for now). People/projects/ideas deferred to v3.2.
- **D-04:** Admin Agent processing trigger moves to the Tasks tab (GET /api/errands on focus). Status tab no longer triggers processing.
- **D-05:** Investigate stays as a push screen from Status (magnifying glass icon), not a tab. 4 tabs is the limit.

### Error Deep-Link
- **D-06:** Error card deep-link passes an explicit time range in the investigation query matching the dashboard's 24h window, so the investigation agent finds the errors.
- **D-07:** Error card shows recency info (e.g., "4 errors — last: 3h ago") so the user knows if errors are fresh before tapping.

### EAS Build Strategy
- **D-08:** Ship a production EAS build submitted to TestFlight for everyday use away from home.
- **D-09:** API URL stays baked into eas.json env vars for all profiles. Only the API key is prompted on first launch.
- **D-10:** API key stored in expo-secure-store (encrypted). Minimal settings screen (gear icon or similar) allows re-entering the key later.
- **D-11:** Set up EAS Update for OTA JS-only pushes — allows quick fixes without full rebuild after TestFlight submission.
- **D-12:** Dev builds continue using env vars for both URL and key (no prompt in development).

### Errands Visibility
- **D-13:** On the Tasks tab, errand sections start collapsed (current behavior preserved).
- **D-14:** Tasks tab empty state: "No tasks" (minimal, matching current style).
- **D-15:** Status screen removes all errand-related content — no fetching, no empty message, no SectionList. Just dashboard cards + spine tiles + investigate icon.

### Claude's Discretion
- Settings screen design and navigation (modal, screen push, or inline)
- Exact investigation query wording for time-ranged error deep-link
- How /api/health-summary returns recency data (new field vs. computed client-side)
- EAS Update channel naming and configuration details
- TestFlight metadata (description, screenshots, etc.)

</decisions>
