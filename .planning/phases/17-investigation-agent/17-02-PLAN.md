<objective>
Build the Investigation Agent: tools, agent registration, SSE streaming adapter, API endpoint, and lifespan wiring.

Purpose: This is the core of Phase 17 -- a third Foundry assistant that answers natural language questions about captures and system health. The agent receives user questions, decides which KQL tools to call, the backend executes those queries via the Plan 01 query functions, returns results to the assistant, and the assistant formats human-readable answers that stream as SSE events. Unlike the Classifier (which suppresses text output), the Investigation Agent's text IS the primary deliverable.

Output: Working POST /api/investigate endpoint that accepts a question, streams SSE events (thinking, tool_call, text, done), and supports multi-turn conversations via client-managed thread_id.

**File 2: backend/src/second_brain/streaming/investigation_adapter.py**

Create `stream_investigation()` async generator function.

CRITICAL DIFFERENCE from Classifier adapter: Text output is the PRIMARY deliverable, NOT reasoning to suppress. Yield text content as `text` SSE events.

```python
async def stream_investigation(
    client: AzureAIAgentClient,
    question: str,
    tools: list,
    thread_id: str | None = None,
    rate_limiter: SoftRateLimiter | None = None,
) -> AsyncGenerator[str, None]:
```

Implementation:
1. If `rate_limiter` and `rate_limiter.check()` returns False, yield a `rate_warning` SSE event: `{"type": "rate_warning", "message": "You're sending queries quickly. Results may be slower."}`. Do NOT block -- just warn.
2. Yield `{"type": "thinking"}` as first event.
3. Build messages and options:
   ```python
   messages = [Message(role="user", text=question)]
   options: ChatOptions = {"tools": tools}
   if thread_id:
       options["conversation_id"] = thread_id
   ```
   NOTE: Do NOT set `tool_choice` to "required". Investigation Agent uses auto tool_choice so it can respond without calling tools (e.g., "thanks", "what can you help with?"). This is per RESEARCH pitfall #6.
4. Stream via `client.get_response(messages=messages, stream=True, options=options)`.
5. Capture `conversation_id` from first update that has it (for thread continuity).
6. For each content in each update:
   - `content.type == "text"` with text: yield `{"type": "text", "content": content.text}` -- this IS the answer.
   - `content.type == "function_call"` with name: yield `{"type": "tool_call", "tool": name, "description": "Querying {name}..."}` where name is mapped to human-friendly description (trace_lifecycle -> "Tracing capture lifecycle...", recent_errors -> "Querying recent errors...", system_health -> "Checking system health...", usage_patterns -> "Analyzing usage patterns...").
   - `content.type == "function_result"`: tool results go back to assistant automatically via the SDK. No SSE event needed. But if the result contains an error (check for "error" key in parsed JSON), yield a `tool_error` event: `{"type": "tool_error", "tool": tool_name, "error": error_message}`.
7. After stream completes, yield `{"type": "done", "thread_id": conversation_id}` where conversation_id is the captured thread ID (or the original thread_id if none captured).
8. Wrap in `asyncio.timeout(60)`. On timeout or exception, yield `{"type": "error", "message": "Investigation failed. Please try again."}` then `{"type": "done", "thread_id": thread_id or ""}`.
9. Use OTel span `tracer.start_as_current_span("investigate")` with attributes: question length, thread_id, has_tools.

Also create `SoftRateLimiter` class in the same file (or as a separate small module -- executor's choice):
```python
class SoftRateLimiter:
    """Warn-only rate limiter using sliding window."""
    def __init__(self, max_requests: int = 10, window_seconds: int = 60) -> None:
        self._max = max_requests
        self._window = window_seconds
        self._timestamps: deque[float] = deque()
    def check(self) -> bool:
        """Returns True if within limit, False if over (warn only)."""
        now = time.monotonic()
        while self._timestamps and now - self._timestamps[0] > self._window:
            self._timestamps.popleft()
        self._timestamps.append(now)
        return len(self._timestamps) <= self._max
```
