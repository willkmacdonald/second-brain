  <action>
Modify kql_templates.py to:

1. **Enhance SYSTEM_HEALTH** -- Replace the current `toscalar()` + `print` pattern with a `summarize` approach that supports `percentile()`. The enhanced template must:
   - Add `p95_duration_ms = percentile(duration, 95)` and `p99_duration_ms = percentile(duration, 99)` alongside the existing `avg(duration)`
   - Add trend comparison: use two `let` statements for current period and previous period (current = `ago({time_range})`, previous = `between(ago(2*{time_range}) .. ago({time_range}))`)
   - Return both current and previous period metrics in separate rows or columns for comparison
   - Parameterize with `{time_range}` (e.g., "24h", "7d") via str.format() -- the existing template has no time_range parameter (it relies on timespan)
   - NOTE: Since KQL `ago()` requires KQL duration literals like `24h` or `7d`, and `str.format()` is used for parameterization, the template should use `{time_range}` placeholders that accept KQL duration strings directly (e.g., `ago({time_range})`)
   - Keep workspace schema (requests, traces, NOT AppRequests, AppTraces)
   - Rename to `SYSTEM_HEALTH_ENHANCED` so the existing template remains available as fallback. The old SYSTEM_HEALTH can stay as-is.

2. **Enhance RECENT_FAILURES** -- Create `RECENT_FAILURES_FILTERED` that adds:
   - `{component_filter}` placeholder -- when component is specified, adds `| where customDimensions.component == "{component}"` filter; when null, no filter applied
   - `{severity_filter}` placeholder -- minimum severity level (3 for warning, 4 for error)
   - `{limit}` placeholder for row limit (default 10 per CONTEXT.md result cap)
   - Keep original RECENT_FAILURES unchanged as fallback

3. **Add LATEST_CAPTURE_TRACE_ID** -- New template to find the most recent capture trace ID:
   ```
   requests
   | where name has "/api/capture"
   | where toint(resultCode) >= 200 and toint(resultCode) < 400
   | top 1 by timestamp desc
   | extend trace_id = tostring(customDimensions.capture_trace_id)
   | project trace_id, timestamp
   ```

4. **Add USAGE_PATTERNS_BY_PERIOD** -- Captures by time bin:
   ```
   requests
   | where name has "/api/capture"
   | summarize capture_count = count() by bin(timestamp, {bin_size})
   | order by timestamp asc
   ```
   Parameterized with `{bin_size}` (e.g., `1h`, `1d`).

5. **Add USAGE_PATTERNS_BY_BUCKET** -- Bucket distribution:
   ```
   traces
   | where customDimensions.component == "classifier"
   | where message has "Filed to"
   | extend bucket = extract("Filed to (\\w+)", 1, message)
   | where isnotempty(bucket)
   | summarize count_ = count() by bucket
   | order by count_ desc
   ```

6. **Add USAGE_PATTERNS_BY_DESTINATION** -- Destination usage:
   ```
   traces
   | where customDimensions.component == "admin_agent"
   | where message has "Added"
   | extend destination = extract("to ([\\w-]+)", 1, message)
   | where isnotempty(destination)
   | summarize count_ = count() by destination
   | order by count_ desc
   ```

All new templates MUST use workspace schema (traces, requests, dependencies, exceptions) -- never AppTraces, AppRequests.
