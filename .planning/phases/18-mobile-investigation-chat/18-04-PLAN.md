<objective>
Fix dashboard-to-investigation inconsistency where the error card shows agent commentary text that the investigation agent cannot find when queried via deep-link.

Purpose: UAT Test 9 blocker -- dashboard regex-extracts "errors" from agent prose that may be interpretive statements (e.g., "count remained consistent at zero"), not actual Error-level log entries. When the deep-link re-asks about that text, the agent searches recent_errors (AppExceptions + high-severity AppTraces) and finds nothing, responding "No errors found". The root cause is that the dashboard prompt does not force the agent to call the recent_errors tool, so the "error" text comes from a different data scope than what the deep-link query searches.

Output: Revised dashboard health prompt and error regex that ensure consistency between dashboard error display and deep-link investigation.
</objective>
