<objective>
Fix two compounding UAT blockers from Phase 18 UAT tests 8 and 9:

1. **Cross-screen voice event leak** -- The capture screen's global `useSpeechRecognitionEvent` hooks receive events from the investigate screen's voice session. When the investigate screen unmounts and calls `abortRecognition()`, the capture screen's error handler fires `sendVoiceCapture()` with a stale WAV file, which the backend rejects as `audio/vnd.wave`.

2. **Sentry blind spot for caught errors** -- All `console.error()` calls in the mobile codebase lack corresponding `Sentry.captureMessage()` calls, so caught errors never reach Sentry. This means the dashboard "Last Error" card always shows "None" and the investigation agent cannot see client-side errors.

Purpose: Restore the error observability pipeline so UAT tests 8 (dashboard cards) and 9 (error deep-link) pass.
Output: Fixed voice event handling, correct MIME types, Sentry instrumentation on error callbacks.
</objective>

**Fix 3: Fix hardcoded MIME type in ag-ui-client.ts**

In `sendVoiceCapture` (line 316-320), the FormData hardcodes `type: "audio/m4a"` and `name: "voice-capture.m4a"`. The expo-speech-recognition persist feature saves WAV files, not M4A. The cloud fallback `audioRecorder` (expo-audio) saves M4A. The MIME type should reflect the actual file.

Change lines 316-320 to detect format from the URI extension:
```typescript
const isWav = audioUri.toLowerCase().endsWith(".wav");
formData.append("file", {
  uri: audioUri,
  type: isWav ? "audio/wav" : "audio/m4a",
  name: isWav ? "voice-capture.wav" : "voice-capture.m4a",
} as any);
```

This correctly handles both paths:
- Speech recognition persist (WAV) -> `audio/wav`
- Cloud fallback audio recorder (M4A) -> `audio/m4a`

**Fix 4: Add audio/vnd.wave to backend ALLOWED_AUDIO_TYPES**

In `capture.py` line 37-40, add `"audio/vnd.wave"` to the frozenset. This is a valid IANA MIME type for WAV files that iOS may send regardless of what the client specifies in FormData headers:

```python
ALLOWED_AUDIO_TYPES = frozenset({
    "audio/m4a", "audio/mp4", "audio/mpeg", "audio/wav", "audio/vnd.wave",
    "audio/x-m4a", "audio/aac", "audio/ogg",
})
```
