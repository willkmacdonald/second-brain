**Step 2: Create `mobile/lib/investigate-client.ts`.**
Follow the exact pattern from `ag-ui-client.ts` (EventSource with POST, pollingInterval: 0). The function signature:

```typescript
export function sendInvestigation({
  question, threadId, apiKey, callbacks,
}: {
  question: string;
  threadId?: string;
  apiKey: string;
  callbacks: InvestigateCallbacks;
}): { cleanup: () => void }
```

Where `InvestigateCallbacks` has: `onThinking`, `onText(content: string)`, `onDone(threadId: string)`, `onError(message: string)`.

- EventSource type is `"message"` (react-native-sse dispatches as "message" when no `event:` field in SSE).
- Parse JSON from `event.data`, switch on `parsed.type`.
- Handle: `thinking` -> `callbacks.onThinking()`, `text` -> `callbacks.onText(parsed.content ?? "")`, `done` -> `callbacks.onDone(parsed.thread_id ?? "")` + `es.close()`, `error` -> `callbacks.onError(parsed.message ?? "Investigation failed")` + `es.close()`.
- Silently ignore `tool_call`, `tool_error`, `rate_warning` per user decision (no tool call visibility).
- Use explicit null/undefined checks for `thread_id` (not falsy check) -- empty string is falsy in JS (documented pitfall in MEMORY.md and RESEARCH.md Pitfall 3).
- On EventSource error event: extract message, call `callbacks.onError`, close.
- Return `{ cleanup }` that calls `es.removeAllEventListeners()` then `es.close()`.
- Set `pollingInterval: 0` on EventSource constructor (CRITICAL -- prevents auto-reconnection, RESEARCH.md Pitfall 1).

**Step 1: Create `mobile/app/investigate.tsx`.**

Full chat screen with these elements:

**State:**
- `messages: ChatMessage[]` where `ChatMessage = { id: string; role: "user" | "agent"; content: string; isStreaming: boolean }`
- `inputText: string`
- `threadId: string | null` (stored from `done` event, passed on follow-ups)
- `isLoading: boolean` (true while agent is streaming)
- `isRecording: boolean` (voice input state)
- `cleanupRef: React.MutableRefObject<(() => void) | null>` (SSE cleanup)

**Route params:**
- Accept optional `initialQuery` param via `useLocalSearchParams<{ initialQuery?: string }>()`. When present, auto-send the query on mount (for MOBL-06 deep-link from dashboard error card). Use `useEffect` with the param to trigger send once.

**Layout (top to bottom):**
1. `KeyboardAvoidingView` wrapping entire screen, `behavior={Platform.OS === "ios" ? "padding" : "height"}`, `keyboardVerticalOffset` adjusted for header height (~90).
2. FlatList with `inverted={true}` rendering messages in reverse order (newest first). `renderItem` uses `InvestigateBubble`. When `messages` is empty, show `QuickActionChips` as `ListEmptyComponent`.
3. Input bar at bottom: `View` with `TextInput` (placeholder "Ask about your system..."), voice button (microphone icon), and send button (arrow icon). Send button enabled when `inputText.trim()` and not `isLoading`. Voice button toggles recording.

**Send flow (`handleSend` function):**
1. Add user message to `messages` array.
2. Add empty agent message with `isStreaming: true`.
3. Set `isLoading = true`.
4. Call `sendInvestigation({ question, threadId, apiKey: API_KEY, callbacks })`.
5. `onThinking`: no-op (agent message already shows "Thinking..." when content is empty and isStreaming).
6. `onText(content)`: Update the last agent message by appending content to its `content` field. Use functional state update to avoid stale closure.
7. `onDone(newThreadId)`: Set `isStreaming = false` on last agent message, store `threadId` for follow-ups, set `isLoading = false`.
8. `onError(message)`: Set agent message content to error message, `isStreaming = false`, set `isLoading = false`.
9. Store cleanup ref for unmount.

**Quick action chip flow:**
- When `onSelect` fires from `QuickActionChips`, call `handleSend` with the chip's query string. Chips automatically disappear because `messages` is no longer empty.

**Voice input:**
- Declare `useSpeechRecognitionEvent("result", ...)` and `useSpeechRecognitionEvent("end", ...)` hooks at top level of component (RESEARCH.md Pitfall 5 -- hooks must be top-level).
- Import `requestSpeechPermissions`, `startOnDeviceRecognition`, `stopRecognition`, `abortRecognition` from `lib/speech.ts`.
- Voice button press: if not recording, request permissions, start recognition, set `isRecording = true`. If recording, stop recognition, set `isRecording = false`.
- `result` event: update `inputText` with interim/final transcript.
- `end` event: set `isRecording = false`. If final transcript is non-empty, auto-submit by calling `handleSend`. Guard against empty string from expo-speech-recognition stop() (MEMORY.md note).
- Cleanup: `abortRecognition()` on component unmount.

**New chat button:**
- Header right button (refresh/reset icon). On press: clear `messages`, clear `threadId`, clear `inputText`, cleanup any active SSE connection.

**Styling:**
- Container: `flex: 1, backgroundColor: "#0f0f23"`.
- Input bar: `backgroundColor: "#1a1a2e"`, borderTopColor `#333`, padding 8. TextInput: `backgroundColor: "#0f0f23"`, color `#ffffff`, borderRadius 20, paddingHorizontal 16, flex 1.
- Send button: `#4a90d9` background when enabled, `#333` when disabled, borderRadius 20, width 36 height 36.
- Voice button: similar size, show microphone icon, `#ff4444` background when recording.

**Cleanup on unmount:**
- Call `cleanupRef.current?.()` to close SSE connection.
- Call `abortRecognition()` to stop any active voice input.
