<task type="auto">
  <name>Task 1: RBAC role assignment, workspace ID env var, and Cosmos container creation</name>
  <files></files>
  <action>
    All steps use Azure CLI. Run `az account show` first to confirm correct subscription.

    **Step 1: Get the Log Analytics workspace resource ID from App Insights**
    ```bash
    WORKSPACE_RESOURCE_ID=$(az monitor app-insights component show \
        --app second-brain-insights \
        --resource-group shared-services-rg \
        --query "workspaceResourceId" -o tsv)
    echo "Workspace Resource ID: $WORKSPACE_RESOURCE_ID"
    ```

    **Step 2: Get the workspace GUID (customerId) for LogsQueryClient**
    Extract the workspace name from the resource ID and query it:
    ```bash
    # Parse workspace name from resource ID path
    WORKSPACE_NAME=$(echo "$WORKSPACE_RESOURCE_ID" | rev | cut -d'/' -f1 | rev)
    WORKSPACE_RG=$(echo "$WORKSPACE_RESOURCE_ID" | grep -oP 'resourceGroups/\K[^/]+')

    # Get the workspace GUID (customerId)
    WORKSPACE_ID=$(az monitor log-analytics workspace show \
        --workspace-name "$WORKSPACE_NAME" \
        --resource-group "$WORKSPACE_RG" \
        --query "customerId" -o tsv)
    echo "Workspace ID (GUID): $WORKSPACE_ID"
    ```

    **Step 3: Get Container App managed identity principal ID**
    ```bash
    PRINCIPAL_ID=$(az containerapp show \
        --name second-brain-api \
        --resource-group shared-services-rg \
        --query "identity.principalId" -o tsv)
    echo "Principal ID: $PRINCIPAL_ID"
    ```
    If the result is empty or null, the Container App may use user-assigned identity instead. Check with `--query "identity"` to see the full identity config.

    **Step 4: Assign Log Analytics Reader role**
    ```bash
    az role assignment create \
        --assignee "$PRINCIPAL_ID" \
        --role "Log Analytics Reader" \
        --scope "$WORKSPACE_RESOURCE_ID"
    ```
    If role already exists, az CLI will return an existing assignment (idempotent). Note: RBAC propagation can take up to 5-10 minutes.

    **Step 5: Set LOG_ANALYTICS_WORKSPACE_ID env var on Container App**
    ```bash
    az containerapp update \
        --name second-brain-api \
        --resource-group shared-services-rg \
        --set-env-vars "LOG_ANALYTICS_WORKSPACE_ID=$WORKSPACE_ID"
    ```

    **Step 6: Run Cosmos container creation script**
    Requires COSMOS_ENDPOINT. Get it from the Container App's existing env vars or from the Cosmos account:
    ```bash
    # Get Cosmos endpoint from existing config
    COSMOS_ENDPOINT=$(az cosmosdb show \
        --name <cosmos-account-name> \
        --resource-group shared-services-rg \
        --query "documentEndpoint" -o tsv)

    # Or use the value from the Container App's env vars
    export COSMOS_ENDPOINT="<value>"

    cd backend && python3 scripts/create_eval_containers.py
    ```
    Script is idempotent -- safe to re-run if containers already exist.

    **Step 7: Push to main to trigger CI/CD deployment**
    The updated backend code (Plans 01-02) needs to be deployed. Push to main branch. GitHub Actions CI/CD will build the Docker image and deploy to Azure Container Apps.
  </action>
