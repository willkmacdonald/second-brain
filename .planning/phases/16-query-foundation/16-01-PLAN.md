       - `execute_kql(client, workspace_id, query, timespan=timedelta(hours=24)) -> QueryResult`: Core function. Calls `client.query_workspace()` with `server_timeout=60`. Checks `response.status` for `LogsQueryStatus.SUCCESS` vs `LogsQueryStatus.PARTIAL`. For SUCCESS: parses `response.tables` into list of list[dict] (each table -> list of row dicts via `dict(zip(table.columns, row))`). For PARTIAL: parses `response.partial_data` similarly, sets `is_partial=True`, captures `str(response.partial_error)`. Returns `QueryResult`.
       - `query_capture_trace(client, workspace_id, trace_id) -> list[TraceRecord]`: Formats `CAPTURE_TRACE` template with trace_id, calls `execute_kql`, parses first table rows into `TraceRecord` models.
       - `query_recent_failures(client, workspace_id) -> list[FailureRecord]`: Calls `execute_kql` with `RECENT_FAILURES` template, parses into `FailureRecord` models.
       - `query_system_health(client, workspace_id) -> HealthSummary`: Calls `execute_kql` with `SYSTEM_HEALTH` template, parses single summary row into `HealthSummary`.
       - `query_admin_audit(client, workspace_id) -> list[AdminAuditRecord]`: Calls `execute_kql` with `ADMIN_AUDIT_LOG` template, parses into `AdminAuditRecord` models.

       All functions are `async`. All use `logging.getLogger(__name__)` for error/warning logs. All handle empty result sets gracefully (return empty list or HealthSummary with zeros).
