def _response_needs_delivery(response_text: str | None) -> bool:
    # Returns True for rule management feedback, conflict confirmations, query answers

    delivery_indicators = [
        "conflict", "currently goes to", "currently routes to",
        "where should", "which destination", "affinity rule",
        "routing rule", "goes to", "routed to", "renamed",
        "removed destination", "no matching rule",
    ]
    return any(indicator in text_lower for indicator in delivery_indicators)

**2. _response_needs_delivery (admin_handoff.py):**
Add recipe success summary detection to the delivery_indicators list. The Admin Agent will return summaries like "Added 8 items to 2 destinations from Chicken Tikka Masala" after processing a recipe URL. Add these indicators:
```python
"items added",       # Recipe success: "Added 8 items..."
"from",              # Recipe attribution: "...from Chicken Tikka Masala"
"no recipe found",   # Recipe failure: "No recipe found on this page"
"error fetching",    # Fetch failure: "Error fetching https://..."
```

IMPORTANT: The "from" indicator is too broad on its own -- it would match many innocuous responses. Instead, add a compound check: add `"items added"` and `"no recipe found"` and `"error fetching"` to the delivery_indicators list. The "from" keyword combined with "items added" will be naturally present in recipe summaries. Do NOT add bare "from" as an indicator.

So add to `delivery_indicators`:
```python
"items added",        # Recipe success summaries
"no recipe found",    # Recipe extraction failure
"error fetching",     # URL fetch failure
```
