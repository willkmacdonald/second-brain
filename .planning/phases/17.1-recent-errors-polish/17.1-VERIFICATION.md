### Direct comparison to original Test 2 failure

The same `investigation_agent` errors that were visible in the original Phase 17 Test 2 (the one that surfaced the bugs) now render correctly:
- **Before:** Component column rendered as blank, Trace ID column rendered as blank — ambiguous, looked like broken data
- **After:** Component renders `investigation_agent` (because those log entries carry the component tag via `logger.info(..., extra={"component": "investigation_agent"})`), Trace ID renders `N/A` (because they're not part of a capture flow, so no `capture_trace_id`)

The "blank cells" bug from Test 2 is gone. This is direct, observational proof that the Pydantic null-normalization validator + the portal rendering rules work together end-to-end.
