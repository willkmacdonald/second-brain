# Phase 20 — UI Design Contract

**Scope:** Thumbs up/down buttons on inbox item detail modal. This is the only UI change in Phase 20 -- all other work (implicit signals, investigation agent tools, golden dataset promotion) is backend-only or conversational.

## Component Inventory

### New: FeedbackButtons

**Location:** Inside the inbox detail card modal (inbox.tsx), between the Timestamp section and the bucket section.

**Layout:**
```
[Timestamp row]
                                          
FEEDBACK (label)
[ thumbs-down ]  [ thumbs-up ]           

FILE TO BUCKET (label)
[People] [Projects] [Ideas] [Admin]      

[Close]                                   
```

**Component structure (not a separate file):** Inline in the detail modal, following existing patterns. Two `Pressable` buttons in a horizontal `View` row.

**Visual specification:**

| Property | Thumbs Down | Thumbs Up |
|----------|-------------|-----------|
| Icon character | `"\uD83D\uDC4E"` (unicode thumbs down) | `"\uD83D\uDC4D"` (unicode thumbs up) |
| Icon size | 22px | 22px |
| Button width | 56px | 56px |
| Button height | 44px (Apple HIG minimum) | 44px (Apple HIG minimum) |
| Border radius | 12px | 12px |
| Default background | `#2a2a4e` | `#2a2a4e` |
| Active background (selected) | `#ff6b6b` (at 20% opacity: `rgba(255,107,107,0.2)`) | `#4ade80` (at 20% opacity: `rgba(74,222,128,0.2)`) |
| Active border | 1px solid `#ff6b6b` | 1px solid `#4ade80` |
| Default border | none (0px) | none (0px) |
| Pressed opacity | 0.7 | 0.7 |
| Alignment | center (both axes) | center (both axes) |

**Row layout:**
- `flexDirection: "row"`
- `gap: 12`
- `marginTop: 4` (below the FEEDBACK label)
- Buttons are NOT flex:1 -- fixed width 56px each, left-aligned in the row

**Label above buttons:**
- Uses existing `detailLabel` style: fontSize 11, fontWeight 600, color `#666`, textTransform uppercase, letterSpacing 0.5, marginTop 12, marginBottom 4
- Copy: `"Feedback"`

**Interaction behavior:**
1. Both buttons start in default (unset) state when modal opens
2. Tapping thumbs-up: button transitions to active-positive state, thumbs-down stays default
3. Tapping thumbs-down: button transitions to active-negative state, thumbs-up stays default
4. Tapping the already-active button: deselects it (returns to default) -- toggle behavior
5. Switching from one to the other: previous deselects, new one activates
6. On selection, fire-and-forget POST to feedback API (no loading spinner, no blocking)
7. If item already has feedback from a prior session: pre-populate the active state on modal open (requires backend returning feedback status on inbox items -- OPTIONAL, can defer to follow-up)

**State management:**
- Local state `feedbackState: "none" | "thumbs_up" | "thumbs_down"` per modal open
- Reset to `"none"` when modal closes and reopens for a different item

## Copywriting Contract

| Element | Copy |
|---------|------|
| Feedback section label | `FEEDBACK` |
| Empty state (no inbox items) | `No captures yet` / `Tap Capture to add your first thought` (existing -- no change) |
| Feedback toast (thumbs up sent) | `Feedback recorded` |

Note: There is no separate "feedback signals" screen or empty state for feedback data. Feedback is collected inline on inbox items. Signal review and golden dataset promotion happen through the investigation agent chat (D-04), which already has its own empty state.

## Implementation Notes

1. **No new component file needed.** The feedback buttons are 15-20 lines of JSX added inline to the existing detail modal in `inbox.tsx`, following the same pattern as the bucket buttons section. Creating a separate `FeedbackButtons.tsx` component is acceptable but not required given the minimal scope.

2. **No new dependencies.** Uses existing React Native Pressable, existing toast pattern, existing StyleSheet approach.

3. **Backend API call pattern:** Match existing `handleRecategorize` fire-and-forget style. POST to a new endpoint (e.g., `POST /api/feedback`) with `{ inboxItemId, signalType: "thumbs_up"|"thumbs_down", captureText, originalBucket }`. Failure logged via `reportError()`, no user-visible error.

4. **Modal detail card height:** Adding the feedback row increases modal content by approximately 60px (12px label margin + 16px label + 4px gap + 44px buttons). The existing `maxHeight: "70%"` should accommodate this without scrolling for most items.
