<task type="auto">
  <name>Task 2: Add commit-correlated revision naming to deploy step</name>
  <files>.github/workflows/deploy-backend.yml</files>
  <action>
  Modify the existing "Build and push image" and "Deploy to Container Apps" steps:

  1. In the **Build and push image** step, add an output that exports `IMAGE_TAG` to `$GITHUB_ENV` so subsequent steps can reference it:
     ```yaml
     - name: Build and push image
       run: |
         IMAGE_TAG="${{ env.ACR_NAME }}.azurecr.io/${{ env.IMAGE_NAME }}:${{ github.sha }}"
         echo "IMAGE_TAG=$IMAGE_TAG" >> "$GITHUB_ENV"
         docker build --platform linux/amd64 -t "$IMAGE_TAG" ./backend
         docker push "$IMAGE_TAG"
     ```

  2. Update the **Deploy to Container Apps** step to use `--revision-suffix` with a `sha-` prefix on the first 7 characters of the git SHA. This ensures the suffix always starts with an alphabetic character (git SHAs can start with digits, which Azure rejects). Also compute and export `REVISION_SUFFIX` and `REVISION_NAME` to `$GITHUB_ENV`:
     ```yaml
     - name: Deploy to Container Apps
       run: |
         REVISION_SUFFIX="sha-$(echo ${{ github.sha }} | cut -c1-7)"
         echo "REVISION_SUFFIX=$REVISION_SUFFIX" >> "$GITHUB_ENV"
         echo "REVISION_NAME=${{ env.CONTAINER_APP_NAME }}--${REVISION_SUFFIX}" >> "$GITHUB_ENV"
         az containerapp update \
           --name ${{ env.CONTAINER_APP_NAME }} \
           --resource-group ${{ env.RESOURCE_GROUP }} \
           --image "${{ env.IMAGE_TAG }}" \
           --revision-suffix "$REVISION_SUFFIX"
       ```

  The revision name format will be: `second-brain-api--sha-{7chars}` (e.g., `second-brain-api--sha-abc1234`), enabling easy correlation with `git log --oneline`.
  </action>

  </verify>
  <done>Deploy step uses --revision-suffix with sha- prefix, IMAGE_TAG/REVISION_SUFFIX/REVISION_NAME are exported to GITHUB_ENV for Plan 02's verification and summary steps</done>
