# Phase 16.1: Improve Deployment Process - Context

**Gathered:** 2026-04-06
**Status:** Ready for planning

<domain>
## Phase Boundary

Harden the GitHub Actions deploy-backend workflow so deployments are verified, revision state is clean, dependency drift is caught before build, and deploy results are visible in the job log. All changes are to `.github/workflows/deploy-backend.yml` and the deploy pipeline — no application code changes.

</domain>

<decisions>
## Implementation Decisions

### Post-deploy verification
- Poll health endpoint (`https://brain.willmacdonald.com/health`) every 10-15 seconds, 3-minute timeout
- Verify JSON response body contains `{"status":"ok"}` (not just HTTP 200)
- Check that the latest revision has 100% traffic (tolerate Azure's transient activation behavior)
- Verify deployed image SHA matches the git SHA from the build step
- On verification failure: fail the workflow run, no auto-rollback

### Revision management
- After successful deploy + verification: deactivate all revisions except the latest
- No auto-rollback on failure — just fail the workflow, investigate manually
- No pre-deploy health check — just deploy
- Use `--revision-suffix` with first 7 chars of git SHA for commit-correlated revision naming

### Dependency safety
- Add `uv lock --check` as the first step after checkout (within the build job, not a separate job)
- Pin uv version to match Dockerfile (0.5.4) to prevent lockfile format mismatches
- Main branch only — no PR build checks

### Deploy visibility
- GitHub Actions notifications only (no Slack/external integrations)
- Structured summary block at end of job log: revision name, image SHA, health status, active revision count
- Track and log total pipeline duration and per-step timing
- Include clickable link to Container App revisions page in Azure Portal

### Claude's Discretion
- Exact polling interval within the 10-15s range
- Shell scripting approach for the verification steps
- Summary block formatting
- How to construct the Azure Portal URL

</decisions>
