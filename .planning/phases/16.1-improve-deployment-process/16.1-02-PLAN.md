  <action>
  Add two new steps after the existing "Deploy to Container Apps" step:

  1. **Verify deployment health** step: A shell polling loop that checks the health endpoint every 12 seconds for up to 15 attempts (180 seconds = 3 minutes). The step:
     - Curls `https://brain.willmacdonald.com/health`, capturing HTTP status code and response body
     - On HTTP 200, parses the JSON body with `jq` and checks that `.status == "ok"`
     - On success: prints the health response body, sets `HEALTH_STATUS=healthy` in `$GITHUB_ENV`
     - On failure after all attempts: uses `::error::` annotation and `exit 1`
     - Each attempt logs: `Attempt N/15: HTTP {code}` or `Attempt N/15: HTTP 200 but status={status}`

     ```yaml
     - name: Verify deployment health
       run: |
         HEALTH_URL="https://brain.willmacdonald.com/health"
         MAX_ATTEMPTS=15
         POLL_INTERVAL=12

         for i in $(seq 1 $MAX_ATTEMPTS); do
           HTTP_CODE=$(curl -s -o /tmp/health.json -w "%{http_code}" "$HEALTH_URL" 2>/dev/null || echo "000")
           if [ "$HTTP_CODE" = "200" ]; then
             STATUS=$(jq -r '.status' /tmp/health.json)
             if [ "$STATUS" = "ok" ]; then
               echo "Health check passed on attempt $i"
               jq . /tmp/health.json
               echo "HEALTH_STATUS=healthy" >> "$GITHUB_ENV"
               exit 0
             fi
             echo "Attempt $i/$MAX_ATTEMPTS: HTTP 200 but status=$STATUS"
           else
             echo "Attempt $i/$MAX_ATTEMPTS: HTTP $HTTP_CODE"
           fi
           if [ "$i" -eq "$MAX_ATTEMPTS" ]; then
             echo "::error::Health check failed after $MAX_ATTEMPTS attempts (${MAX_ATTEMPTS}x${POLL_INTERVAL}s)"
             echo "HEALTH_STATUS=failed" >> "$GITHUB_ENV"
             exit 1
           fi
           sleep $POLL_INTERVAL
         done
     ```

  2. **Verify deployed image** step: Confirms the deployed revision is running the expected Docker image and checks traffic weight.
     - Uses `az containerapp revision show` to get the deployed image from the revision's container spec
     - Compares against `$IMAGE_TAG` (set by Plan 01)
     - If mismatch: `::error::` annotation and `exit 1`
     - Uses `az containerapp ingress traffic show` to check traffic weight on the latest revision
     - If not 100%: `::warning::` annotation (does NOT fail -- Azure has transient activation behavior)
     - Exports `TRAFFIC_WEIGHT` to `$GITHUB_ENV`

     ```yaml
     - name: Verify deployed image
       run: |
         DEPLOYED_IMAGE=$(az containerapp revision show \
           --name ${{ env.CONTAINER_APP_NAME }} \
           --resource-group ${{ env.RESOURCE_GROUP }} \
           --revision "$REVISION_NAME" \
           --query "properties.template.containers[0].image" \
           -o tsv)

         if [ "$DEPLOYED_IMAGE" != "$IMAGE_TAG" ]; then
           echo "::error::Image mismatch! Expected: $IMAGE_TAG, Got: $DEPLOYED_IMAGE"
           exit 1
         fi
         echo "Image verified: $DEPLOYED_IMAGE"

         TRAFFIC_WEIGHT=$(az containerapp ingress traffic show \
           --name ${{ env.CONTAINER_APP_NAME }} \
           --resource-group ${{ env.RESOURCE_GROUP }} \
           --query "[?revisionName=='$REVISION_NAME'].weight | [0]" \
           -o tsv 2>/dev/null || echo "unknown")

         echo "TRAFFIC_WEIGHT=$TRAFFIC_WEIGHT" >> "$GITHUB_ENV"

         if [ "$TRAFFIC_WEIGHT" != "100" ]; then
           echo "::warning::Latest revision traffic weight is ${TRAFFIC_WEIGHT}%, expected 100%"
         else
           echo "Traffic weight: 100%"
         fi
     ```

<task type="auto">
  <name>Task 2: Add revision cleanup and deploy summary</name>
  <files>.github/workflows/deploy-backend.yml</files>
  <action>
  Add two final steps after the "Verify deployed image" step:

  1. **Cleanup old revisions** step: Deactivates all active revisions except the latest. This prevents revision accumulation (Azure has a 100-revision cap).
     - Lists all active revisions with `az containerapp revision list` using JMESPath query
     - Iterates and deactivates any revision whose name does not match `$REVISION_NAME`
     - Counts active revisions before and after cleanup
     - Exports `ACTIVE_COUNT` (post-cleanup count) to `$GITHUB_ENV`

     ```yaml
     - name: Cleanup old revisions
       run: |
         ACTIVE_REVISIONS=$(az containerapp revision list \
           --name ${{ env.CONTAINER_APP_NAME }} \
           --resource-group ${{ env.RESOURCE_GROUP }} \
           --query "[?properties.active==\`true\`].name" \
           -o tsv)

         BEFORE_COUNT=0
         for rev in $ACTIVE_REVISIONS; do
           BEFORE_COUNT=$((BEFORE_COUNT + 1))
         done
         echo "Active revisions before cleanup: $BEFORE_COUNT"

         DEACTIVATED=0
         for rev in $ACTIVE_REVISIONS; do
           if [ "$rev" != "$REVISION_NAME" ]; then
             echo "Deactivating: $rev"
             az containerapp revision deactivate \
               --revision "$rev" \
               --resource-group ${{ env.RESOURCE_GROUP }} 2>/dev/null || echo "Warning: failed to deactivate $rev"
             DEACTIVATED=$((DEACTIVATED + 1))
           fi
         done

         ACTIVE_COUNT=$((BEFORE_COUNT - DEACTIVATED))
         echo "ACTIVE_COUNT=$ACTIVE_COUNT" >> "$GITHUB_ENV"
         echo "Deactivated $DEACTIVATED revision(s), $ACTIVE_COUNT active"
     ```

  2. **Deploy summary** step with `if: always()` so it runs even when prior steps fail. Writes a structured markdown table to `$GITHUB_STEP_SUMMARY`. Includes:
     - Revision name
     - Image tag
     - Health status (healthy/failed/skipped)
     - Traffic weight
     - Active revision count
     - Total pipeline duration (from PIPELINE_START)
     - Clickable link to Azure Portal Container App revisions page (tenant-agnostic URL using resource ID from `az containerapp show`)

     ```yaml
     - name: Deploy summary
       if: always()
       run: |
         PIPELINE_END=$(date +%s)
         DURATION=$((PIPELINE_END - ${PIPELINE_START:-$PIPELINE_END}))

         SUBSCRIPTION_ID=$(az account show --query id -o tsv 2>/dev/null || echo "unknown")
         PORTAL_URL="https://portal.azure.com/#@willmacdonald.com/resource/subscriptions/${SUBSCRIPTION_ID}/resourceGroups/${{ env.RESOURCE_GROUP }}/providers/Microsoft.App/containerApps/${{ env.CONTAINER_APP_NAME }}/revisionManagement"

         cat >> "$GITHUB_STEP_SUMMARY" << EOF
         ## Deploy Summary

         | Property | Value |
         |----------|-------|
         | Revision | \`${REVISION_NAME:-unknown}\` |
         | Image | \`${IMAGE_TAG:-unknown}\` |
         | Health | ${HEALTH_STATUS:-skipped} |
         | Traffic | ${TRAFFIC_WEIGHT:-unknown}% |
         | Active Revisions | ${ACTIVE_COUNT:-unknown} |
         | Duration | ${DURATION}s |
         | Portal | [View Revisions](${PORTAL_URL}) |
         EOF
     ```

  Ensure the `if: always()` condition is set on the summary step so deploy failures still produce a visible summary.
  </action>

  </verify>
