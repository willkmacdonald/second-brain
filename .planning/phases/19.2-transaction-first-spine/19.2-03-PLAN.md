<objective>
Add backend read-paths the web UI will consume for the ledger-first segment page and the cross-segment transaction page. Specifically:
1. A new `SpineRepository.get_recent_transaction_events` method that returns correlated workload events for a segment, newest-first, with a `limit` cap. Uncorrelated health/probe noise does not belong in the transaction ledger.
2. A new `TransactionLedgerRow` Pydantic model shape carrying everything CONTEXT.md requires (stage/segment_id, timestamp, duration, outcome, correlation_id, operation, error_class, payload summary), plus `SegmentLedgerResponse` metadata telling the UI whether the segment is `transactional` or `native_only`.
3. A new `backend/src/second_brain/spine/ledger_policy.py` module that codifies the operator-facing chain expectations and segment modes, including the approved mobile push-path decision from `SPIKE-MEMO.md`.
4. A new `GET /api/spine/ledger/segment/{id}` route returning transaction rows plus purposeful empty-state metadata for a segment.
5. A new `GET /api/spine/ledger/correlation/{kind}/{id}` route returning the full cross-segment event stream for one correlation id, enriched with duration and operation from the raw `spine_events` join (Option A from RESEARCH §Pitfall 4), AND containing `missing_required` / `present_optional` / `unexpected` segment lists derived from `LEDGER_EXPECTED_CHAINS` so the web UI can render explicit gaps without false mobile alarms.

Purpose: The existing `SegmentDetailResponse` and `CorrelationResponse` shapes are under-specified for CONTEXT's "stage / timestamp / duration / outcome / correlation id / payload summary" requirement. Plans 04/05 cannot render the ledger UI without these shapes. More importantly, the first drill-down must answer "what transactions touched this segment?" rather than "how many health probes arrived?" So the ledger read model must filter to transaction-like, correlated events and carry enough metadata for purposeful empty states. The correlation stream still joins `spine_correlation` (which-segments) with `spine_events` (duration/operation) per RESEARCH Option A — lower risk than a forward-only schema change.

Output: Two new endpoints under `/api/spine/ledger/*`, enriched models, storage method, and tests. All existing spine routes and models unchanged.

```python
async def get_recent_transaction_events(
    self,
    segment_id: str,
    window_seconds: int,
    limit: int = 50,
) -> list[dict[str, Any]]:
    """Return correlated workload events for a segment, newest-first, capped at `limit`.

    This is the transaction/noise boundary for the operator UI:
    uncorrelated workload rows (for example GET /health probes) stay in native
    diagnostics and DO NOT appear in the ledger-first section.
    """
    cutoff = (datetime.now(UTC) - timedelta(seconds=window_seconds)).isoformat()
    results: list[dict[str, Any]] = []
    async for item in self._events.query_items(
        query=(
            "SELECT * FROM c"
            " WHERE c.segment_id = @sid"
            " AND c.timestamp >= @cutoff"
            " AND c.event_type = 'workload'"
            " AND IS_DEFINED(c.payload.correlation_kind)"
            " AND NOT IS_NULL(c.payload.correlation_kind)"
            " AND IS_DEFINED(c.payload.correlation_id)"
            " AND NOT IS_NULL(c.payload.correlation_id)"
            " ORDER BY c.timestamp DESC"
        ),
        parameters=[
            {"name": "@sid", "value": segment_id},
            {"name": "@cutoff", "value": cutoff},
        ],
    ):
        results.append(item)
        if len(results) >= limit:
            break
    return results


async def get_workload_events_for_correlation(
    self,
    correlation_kind: CorrelationKind,
    correlation_id: str,
    window_seconds: int,
) -> list[dict[str, Any]]:
    """Return raw workload events matching a correlation_id within the window.

    Used to enrich CorrelationEvents with duration/operation. The
    spine_correlation container has segment/status/headline but not duration
    or operation; those live on the raw event row in spine_events.
    """
    cutoff = (datetime.now(UTC) - timedelta(seconds=window_seconds)).isoformat()
    results: list[dict[str, Any]] = []
    async for item in self._events.query_items(
        query=(
            "SELECT * FROM c"
            " WHERE c.event_type = 'workload'"
            " AND c.timestamp >= @cutoff"
            " AND c.payload.correlation_kind = @kind"
            " AND c.payload.correlation_id = @cid"
        ),
        parameters=[
            {"name": "@cutoff", "value": cutoff},
            {"name": "@kind", "value": correlation_kind},
            {"name": "@cid", "value": correlation_id},
        ],
    ):
        results.append(item)
    return results
```

Route 2 — cross-segment transaction path with explicit gap reporting:
```python
@router.get(
    "/ledger/correlation/{kind}/{correlation_id}",
    response_model=TransactionPathResponse,
    dependencies=[Depends(auth_dependency)],
)
async def transaction_path(
    kind: CorrelationKind,
    correlation_id: str,
    window_seconds: int = Query(86400, ge=60, le=604800),  # 1min - 7d for enrichment lookup
) -> TransactionPathResponse:
    start = time.perf_counter()
    # "Which segments" — from spine_correlation (ordered ASC by timestamp)
    corr_rows = await repo.get_correlation_events(kind, correlation_id)
    # "Operation/duration enrichment" — from spine_events filtered by correlation
    raw_events = await repo.get_workload_events_for_correlation(
        correlation_kind=kind,
        correlation_id=correlation_id,
        window_seconds=window_seconds,
    )
    # Index raw by (segment_id, timestamp) for enrichment join
    raw_by_segment_ts: dict[tuple[str, str], dict[str, Any]] = {
        (e["segment_id"], e["timestamp"]): e for e in raw_events
    }
    events: list[TransactionEvent] = []
    for cr in corr_rows:
        key = (cr["segment_id"], cr["timestamp"])
        raw = raw_by_segment_ts.get(key)
        payload = raw.get("payload", {}) if raw else {}
        events.append(TransactionEvent(
            segment_id=cr["segment_id"],
            timestamp=parse_cosmos_ts(cr["timestamp"]),
            status=cr["status"],
            headline=cr["headline"],
            operation=payload.get("operation"),
            outcome=payload.get("outcome"),
            duration_ms=payload.get("duration_ms"),
            error_class=payload.get("error_class"),
        ))
    # Expected-chain set math — use the operator-facing ledger policy, not the raw audit registry
    chain = LEDGER_EXPECTED_CHAINS[kind]
    chain_ids = {s.segment_id for s in chain}
    required = {s.segment_id for s in chain if s.required}
    optional = {s.segment_id for s in chain if not s.required}
    segments_seen = {e.segment_id for e in events}
    missing_required = sorted(required - segments_seen)
    present_optional = sorted(optional & segments_seen)
    unexpected = sorted(segments_seen - chain_ids)
    latency_ms = int((time.perf_counter() - start) * 1000)
    return TransactionPathResponse(
        correlation_kind=kind,
        correlation_id=correlation_id,
        events=events,
        missing_required=missing_required,
        present_optional=present_optional,
        unexpected=unexpected,
        envelope=ResponseEnvelope(
            generated_at=datetime.now(UTC),
            freshness_seconds=0,
            partial_sources=[],
            query_latency_ms=latency_ms,
        ),
    )
```
