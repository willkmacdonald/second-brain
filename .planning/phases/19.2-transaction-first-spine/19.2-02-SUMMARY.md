## Accomplishments

- **One-line fix at `agent_emitter.py:63` repairs three segments.** Classifier, admin, and investigation all route through `emit_agent_workload` via `stream_wrapper.py:41` / `admin_handoff.py:397` / `investigate.py:87`. Wrapping the concrete `_WorkloadEvent` in `IngestEvent(root=event)` before `repo.record_event(...)` means `SpineRepository.record_event`'s `event.root` accessor no longer raises `AttributeError`. All three emit sites are fixed in a single commit (`b6394c7`).
- **Mobile crud latent bug fixed.** `api/telemetry.py:105` (workload emit) and `:120` (liveness emit) had the same shape bug. Fixed with identical `IngestEvent(root=...)` wraps. The latent path only fires on mobile CRUD failure, but now it will work the first time it's exercised instead of silently swallowing the AttributeError (`c23befe`).
- **External_services recipe path migrated to the shared helper.** Direct `_WorkloadEvent` construction in `tools/recipe.py:175-185` is replaced with a call to `emit_agent_workload`. The helper centralises the `IngestEvent` wrap AND threads correlation from `capture_trace_id_var` — which the classifier adapter already sets before invoking @tool calls. Recipe external_services rows will now land with `correlation_kind="capture"` and join `spine_correlation` instead of being orphaned from the capture chain. The `except Exception: pass` that silently swallowed emit failures is replaced by `emit_agent_workload`'s own `logger.warning(..., exc_info=True)` path (`678d7fb`).
- **Mobile Option B push path implemented correctly.** The emit is centralised inside `attachCallbacks` (not per-sendX wrappers), fires exactly once per capture via a closure-scoped `emitted` boolean, and covers all 10 terminal paths from memo §4c (CLASSIFIED, COMPLETE/RUN_FINISHED non-HITL, MISUNDERSTOOD, LOW_CONFIDENCE, UNRESOLVED, ERROR, RUN_ERROR, SSE transport error, legacy CUSTOM HITL_REQUIRED, legacy CUSTOM MISUNDERSTOOD/UNRESOLVED). Outcome mapping (success / degraded / failure) matches §4c verbatim. Body shape conforms to `backend/src/second_brain/spine/api.py:59`'s `IngestEvent` Pydantic RootModel so no shape bug is possible on the mobile side (`d310f93`).
- **CI regression coverage that catches future shape-bug reintroduction.** Three new §5.4 tests: (a) `test_stream_wrapper_emit_uses_ingest_event_wrap` — success path through `_RootAccessingSpineRepo`; (b) `test_stream_wrapper_emits_failure_event_on_error` — failure branch in the wrapper's `finally`; (c) `test_no_bare_workload_event_record_event_calls_remain` — static source scan that fails if anyone writes `record_event(_WorkloadEvent(` / `_LivenessEvent(` / `_ReadinessEvent(` without wrapping. The class of bug the spike identified cannot silently come back (`fdf320e`).
- **Classifier-side emit verification at the correct integration level** — `test_classifier_emit_verification_workload_lands_with_trace` exercises `spine_stream_wrapper` with exactly the `segment_id="classifier"` + `operation="capture_text"` values the production capture handler uses, through a RootAccessingSpineRepo double. Asserts a single workload event lands with correlation correctly after the stream completes (`f9756bc`).
- **Stale pre-existing tests that encoded the broken shape are repaired.** Seven AsyncMock-based tests in `test_spine_agent_emitter.py` (3), `test_spine_stream_wrapper.py` (3), `test_telemetry_crud_failure.py` (2), and `test_recipe_workload_emission.py` (2) were tacitly asserting the raw `_WorkloadEvent` shape. Updated to read through `.root.segment_id` / `.root.payload` to match the post-fix production contract. Classified as Rule 1 deviation because the fixes made them stale by design (`157e3a6` + parts of the fix commits). No behaviour change — same underlying event values are being asserted.
- **Integrated-release checklist pre-populated with Plan 01 baselines.** `SPIKE-INTEGRATED-RELEASE-VERIFY.md` carries the before/after counts verbatim from SPIKE-DATA.md §3, the pass criteria per segment, the audit_correlation invocation template, the App Insights KQL for verifying the AttributeError stacktrace is absent post-deploy, and the explicit "do not merge Plan 02 alone" banner at the top (`bfe33d9`).
