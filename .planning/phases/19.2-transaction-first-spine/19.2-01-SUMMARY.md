## Accomplishments

- **Ground truth collected end-to-end** against `https://brain.willmacdonald.com` using a caller-supplied `X-Trace-Id=spike-20260418T235549Z` as the audit key, plus a parallel thread-chain repro (`spike-thread-20260418T235618Z`). Captured raw output from `spine_events`, `spine_correlation`, App Insights (AppRequests / AppTraces / AppDependencies / AppExceptions), and `audit_correlation` in both targeted and sample mode.
- **Root cause identified in one line of code.** `backend/src/second_brain/spine/agent_emitter.py:63` calls `await repo.record_event(event)` with `event` being a raw `_WorkloadEvent`. `record_event` does `event.root` (Pydantic v2 `RootModel` accessor) which raises `AttributeError: '_WorkloadEvent' object has no attribute 'root'` every call. The same shape bug is latent at `api/telemetry.py:105,120` (mobile crud-failure path) and `tools/recipe.py:185` (external_services). Proof: three `AttributeError` stack traces in `AppExceptions` at 23:55:56 / 23:56:14 / 23:56:18 during the spike window, one per broken segment.
- **Per-segment categorization table** with 9 rows, each exactly one of `broken_emitter | correlation_lost | pull_by_design | working_correctly`:
  - **4 broken_emitter:** classifier, admin, investigation, external_services — all fail at `agent_emitter.py:63` (the first three via `emit_agent_workload`; `external_services` via direct `_WorkloadEvent` at `tools/recipe.py:185`)
  - **1 correlation_lost:** backend_api — emit works (27,639 rows/24h) but native `AppRequests` spans do not carry `capture_trace_id`; reproduces the `audit_correlation` instrumentation_warning
  - **4 pull_by_design:** cosmos, container_app, mobile_ui, mobile_capture (for the pre-Option-B capture path)
  - **0 working_correctly** — backend_api is the closest but falls short on native-side correlation
- **Mobile push-path decision locked:** YES for `mobile_capture`, NO for `mobile_ui`. Plan 02 ships **Option B** (post-capture fire-and-forget) rather than Option A (pre-capture with offline queue) — honest trade-off disclosed in memo §4a; ledger can distinguish downstream failures but cannot prove transport failure, which §4d addresses with a dedicated UI empty-state in Plan 04.
- **Option B mapping is complete:** Memo §4c enumerates all 7 terminal paths in the current `attachCallbacks` helper (plus 3 legacy `CUSTOM` HITL paths) — `CLASSIFIED`, `COMPLETE`/`RUN_FINISHED`, `MISUNDERSTOOD`, `LOW_CONFIDENCE`, `UNRESOLVED`, `ERROR`/`RUN_ERROR`, SSE transport `error` listener — and specifies the outcome mapping (success / degraded / failure) for each. Emit is centralised inside `attachCallbacks` with a single-fire guard. This detail matters because per-`sendX` `onComplete`/`onError` wrappers would miss every HITL path (HITL events set `hitlTriggered=true` which suppresses the downstream `COMPLETE` → `onComplete` dispatch).
- **Plan 02 scope is locked** to the 6 numbered fixes in §5 of the memo, each mapped to a specific `file:line`. The categorization and citations are defensible against the `audit_correlation` report (no contradictions).

## Plan 02 Scope Callout

Plan 02 addresses the **4 `broken_emitter` segments** (classifier, admin, investigation, external_services) and the **`mobile_capture` push-path addition** with these fixes, verbatim from memo §5:

1. `backend/src/second_brain/spine/agent_emitter.py:63` — wrap `event` in `IngestEvent(root=event)` before `record_event`. Single edit repairs classifier + admin + investigation simultaneously.
2. `backend/src/second_brain/api/telemetry.py:105,120` — same wrap for both the workload emit (line 105) and liveness emit (line 120).
3. `backend/src/second_brain/tools/recipe.py:175,185` — migrate to `emit_agent_workload` (or wrap); thread `capture_trace_id` into `WorkloadPayload` so correlation rows land; replace bare `except Exception: pass` with `except Exception: logger.warning(..., exc_info=True)`.

## Issues Encountered

- **Round 1 memo under-specified the mobile push-path.** Operator feedback during round 1 review flagged that the decision was stated but the implementation trade-off (pre-capture with queue vs. post-capture fire-and-forget) was not disclosed. Resolved in round 2 (`c66fc48`) with explicit Option A vs. Option B framing and Option B selection justification.
- **Round 2 memo under-specified terminal-path coverage for the mobile emit.** `wrappedCallbacks` around each `sendX` would miss every HITL path because `MISUNDERSTOOD` / `LOW_CONFIDENCE` / `UNRESOLVED` set `hitlTriggered=true` which suppresses the downstream `COMPLETE` → `onComplete` dispatch. Resolved in round 3 (`febcca0`) by enumerating all 7 + 3 legacy terminal paths from the real `attachCallbacks` code, centralising the emit inside `attachCallbacks` itself with a single-fire guard, and documenting the outcome mapping.
- **Operator-cited line numbers needed re-verification across rounds.** Round 2 updated line citations for `agent_emitter.py:63`, `tools/recipe.py:175/185`, `mobile/lib/ag-ui-client.ts:209/215`, and `mobile/lib/telemetry.ts:30` (corrected from an operator-cited "line 83" which doesn't exist in the 51-line file).
