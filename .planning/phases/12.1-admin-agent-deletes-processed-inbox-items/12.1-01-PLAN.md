<objective>
Replace the Admin Agent's "flag as processed" behavior with "delete after processing" so the Inbox stays clean -- users only see unprocessed items.

Purpose: Processed inbox items clutter the Inbox screen. Since shopping list items are the durable output (living in the ShoppingLists container), the inbox item is a transient routing artifact that should be removed once the Admin Agent finishes.

Output: Modified `admin_handoff.py` with delete-on-success and updated tests.
</objective>
