# App Insights KQL Queries

Pre-built queries for investigating Second Brain operational data in Azure Application Insights.

## How to Use

1. Open the Azure Portal: https://portal.azure.com
2. Navigate to **Application Insights** > **second-brain-insights** (in `shared-services-rg`)
3. Click **Logs** in the left sidebar (under Monitoring)
4. Copy-paste the contents of any `.kql` file into the query editor
5. Click **Run**

## Available Queries

| File | Question it Answers | Notes |
|------|-------------------|-------|
| `capture-trace.kql` | What happened to capture X? | Replace `PASTE_TRACE_ID_HERE` with the trace ID from the mobile app |
| `recent-failures.kql` | What failed recently? | Shows ERROR+ logs and exceptions from last 24h |
| `system-health.kql` | How is the system doing overall? | Multi-section: volume, success rate, errors, latency, admin activity |
| `admin-agent-audit.kql` | What did the Admin Agent do? | Processing activity, errand creation, failures |

## Tips

- **capture-trace.kql** requires a trace ID. Find it in the mobile app on the capture detail screen, or in the inbox item details.
- **system-health.kql** has multiple sections. Run the full query for Section 1, or uncomment individual sections and run them separately. Some sections include `| render timechart` hints for visual charts.
- **admin-agent-audit.kql** also has multiple sections. The default query shows all admin activity. Uncomment the other sections for per-capture summaries, errand creation logs, or failures only.
- Data retention is **30 days**. Queries will not return data older than 30 days.

## Trace ID Flow

The `capture_trace_id` is a UUID generated by the mobile app on each capture. It propagates through the entire backend pipeline:

```
Mobile App (generates UUID)
  -> X-Trace-Id header
    -> Backend capture endpoint
      -> Classifier agent
        -> Inbox document (captureTraceId field)
          -> Admin Agent processing
            -> Errand item creation
```

All backend log entries include `capture_trace_id` in their custom dimensions, making it the primary key for investigating any capture's journey.
